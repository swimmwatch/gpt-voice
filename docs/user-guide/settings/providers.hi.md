# प्रदाता सेटिंग्स

GPT-Voice Command Dock में ट्रांसक्रिप्शन प्रदाता चुनें। वर्तमान में चयनित प्रदाता को कॉन्फ़िगर करने के लिए, हेडर में इसका **Connect** या **Configure** नियंत्रण खोलें। प्रदाता संवाद केवल वे नियंत्रण दिखाता है जो उस प्रदाता पर लागू होते हैं।

## ChatGPT Web

ChatGPT Web एपीआई कुंजी के बजाय ब्राउज़र सत्र का उपयोग करता है। इसका प्रदाता संवाद प्रदर्शित करता है कि कोई सत्र सहेजा गया है या नहीं।

1. Command Dock में **ChatGPT Web** चुनें।
2. इसका प्रदाता संवाद खोलें और जब कोई सत्र सहेजा न जाए तो **Log in** चुनें, या सहेजे गए सत्र को बदलने के लिए **Log in again** चुनें।
3. GPT-Voice ब्राउज़र विंडो में साइन-इन पूरा करें, फिर जब प्रदाता **Connected** दिखाता है तो Command Dock पर वापस लौटें।

जब आप सहेजे गए ChatGPT Web प्रमाणीकरण को भूलने के लिए GPT-Voice चाहते हैं तो **Clear session** का उपयोग करें। GPT-Voice इसे साफ़ करने से पहले पुष्टि मांगता है। सत्र साफ़ करने से यह प्रदाता डिस्कनेक्ट हो जाता है; ट्रांसक्रिप्शन के लिए इसका उपयोग करने से पहले दोबारा साइन इन करें।

खाता स्वामित्व, ब्राउज़र-सत्र और प्रदाता-सीमा विवरण के लिए, [प्रदाता](../guides/providers.md) देखें।

## OpenAI API

**OpenAI API** का चयन करें और इन फ़ील्ड को कॉन्फ़िगर करने के लिए इसका प्रदाता संवाद खोलें। अपनी स्वयं की OpenAI API कुंजी और खाते का उपयोग करें; GPT-Voice OpenAI सेवाओं तक कोई कुंजी, क्रेडिट या पहुंच प्रदान नहीं करता है।

| फ़ील्ड          | वर्तमान व्यवहार                                                                                                                                                          |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **API key**     | इसे सहेजने के लिए एक नई कुंजी दर्ज करें। कुंजी संग्रहीत होने पर भी दोबारा खोलने पर फ़ील्ड रिक्त होती है, और रिक्त सहेजें उस संग्रहीत कुंजी को प्रतिस्थापित नहीं करती है। |
| **Model**       | `whisper-1` एकमात्र उपलब्ध ट्रांसक्रिप्शन मॉडल है। इसे केवल पढ़ने योग्य फ़ील्ड के रूप में दिखाया गया है।                                                                 |
| **Language**    | स्वचालित पहचान (डिफ़ॉल्ट), अंग्रेजी, रूसी, यूक्रेनी या बेलारूसी चुनें।                                                                                                   |
| **Prompt**      | वैकल्पिक प्रतिलेखन मार्गदर्शन. डिफ़ॉल्ट खाली है; सहेजे जाने पर अग्रणी और पिछला रिक्त स्थान हटा दिया जाता है।                                                             |
| **Temperature** | 0 से 1 तक प्रतिलेखन भिन्नता को नियंत्रित करता है। डिफ़ॉल्ट 0 है; नियंत्रण 0.05 चरणों में बदलता है।                                                                       |

परिवर्तनों को सत्यापित और संग्रहीत करने के लिए **Save** चुनें। एक सफल सेव संवाद बंद कर देता है। अमान्य मॉडल, भाषा, या तापमान मान अस्वीकार कर दिए जाते हैं; यदि बचत विफल हो जाती है, तो संवाद एक सुरक्षित त्रुटि संदेश दिखाता है और खुला रहता है।

## संग्रहीत क्रेडेंशियल्स और प्रमाणीकरण साफ़ करना

GPT-Voice एक OpenAI API कुंजी को केवल Electron safe storage के माध्यम से संग्रहीत करता है। कुंजी स्वयं संवाद में वापस नहीं दिखाई जाती है; इसके बजाय संवाद इंगित करता है कि एक एपीआई कुंजी संग्रहीत है। यदि सुरक्षित भंडारण अनुपलब्ध है, तो GPT-Voice नई कुंजी सहेज नहीं सकता।**Clear API key** का उपयोग करें और अन्य OpenAI API सेटिंग्स को बनाए रखते हुए संग्रहीत कुंजी को हटाने के लिए संवाद की पुष्टि करें। बटन केवल तभी उपलब्ध होता है जब कोई कुंजी संग्रहीत होती है। OpenAI API दोबारा लिखने से पहले आपको एक नई कुंजी दर्ज करनी होगी और सहेजनी होगी।

प्रदाता की साख और सेवा का उपयोग प्रदाता खाते की शर्तों, बिलिंग, कोटा और गोपनीयता नीति के अंतर्गत रहता है। समर्थन अनुरोधों, स्क्रीनशॉट या दस्तावेज़ में कुंजियाँ न चिपकाएँ।

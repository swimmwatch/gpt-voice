# 設定の概要

GPT-Voice トレイ メニューから **Settings** を開きます。 **Shortcuts** で [設定] ウィンドウが開き、次の 4 つのセクションがあります。

|セクション | | に使用します。
| ------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| **Shortcuts** |文字起こしの録音、停止、キャンセル、翻訳、Prettify、および再試行のためのグローバル ショートカットを選択します。 |
| **[Prettify](prettify.md)** | Ollama または vLLM テキスト処理プロバイダーを選択し、そのモデルと生成動作を構成します。 |
| **[Browser](browser.md)** | GPT-Voice がブラウザベースのサービスに使用するブラウザ ID とバックグラウンド動作を構成します。 |
| **[Network](network.md)** |これらのブラウザベースのサービスで使用されるプロキシを構成します。 |

セクション ボタンは、アクセス可能なラベルが付いたアイコンとして狭いウィンドウでも引き続き使用できます。セクションを選択して設定を変更します。各ページでは、独自のフィールドと前提条件について説明します。

## 変更を保存する

ウィンドウが開くと、保存されたアプリケーション構成から設定がロードされます。値を変更すると、フォームは **Unsaved changes** を持つものとしてマークされます。 **Save changes** ボタンは、少なくとも 1 つの変更が行われ、現在のフィールド値がすべて有効になった後にのみ使用できます。 GPT-Voice の保存中は無効のままです。

値が要件を満たしていない場合、影響を受けるフィールドに検証メッセージが表示され、修正するまで保存がブロックされます。保存操作自体が失敗した場合、設定は開いたままになり、エラー メッセージが表示されるため、問題を修正するか、再試行できます。保存が成功すると、保存された構成が更新され、[設定] ウィンドウが閉じます。

## 誤って作業を失わずに閉じる

変更を保存せずに設定を閉じると、すぐに閉じられます。未保存の変更がある場合、GPT-Voice はそれらを破棄するかどうかを尋ねます。 **Keep editing** を選択してフォームに戻るか、**Discard changes** を選択して保留中の編集を保存せずに設定を閉じます。保存の進行中は、操作が終了するまで設定を閉じることはブロックされます。

この確認は設定フォームに適用されます。新しいグローバル ショートカットのキャプチャは別のアクションです。 GPT-Voice は、キーの組み合わせをリッスンしている間、グローバル ショートカットを一時的に停止し、キャプチャが終了するかキャンセルされると再開します。

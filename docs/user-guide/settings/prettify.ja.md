# プリティファイ設定

**Prettify** は、選択されたテキストに対する GPT-Voice の意味を保持するクリーンアップ アクションです。 **Settings** を開き、**Prettify** を選択して、サービス、そのモデル、およびその生成動作を選択します。テキストを選択してアクションを開始する方法については、[選択したテキストの翻訳と整形](../guides/text-actions.md)を参照してください。

Prettify には、運営する Ollama または vLLM サービスと、そのサービス用に選択されたモデルが必要です。 GPT-Voice は、どちらのサービスもダウンロード、開始、ホストしたり、料金を支払ったりすることはありません。モデルフィールドは必須です。モデルが設定されていない場合、Prettify は選択されたテキストを送信する代わりにモデルが必要であることを報告します。

## プロバイダーを選択して接続します

**Provider** で **Ollama** または **vLLM** を選択します。 GPT-Voice は、ベース URL と選択されたモデルをプロバイダーごとに個別に保持するため、プロバイダーを切り替えても、他のプロバイダーの選択が置き換えられることはありません。デフォルトのプロバイダーは Ollama です。

| プロバイダー | デフォルトのベース URL     | モデルのデフォルト         | 接続動作                                                                                    |
| ------------ | -------------------------- | -------------------------- | ------------------------------------------------------------------------------------------- |
| **Ollama**   | `http://127.0.0.1:11434`   | モデルが選択されていません | Ollama サービスから利用可能なモデルを更新し、それに Prettify リクエストを送信します。       |
| **vLLM**     | `http://127.0.0.1:8000/v1` | モデルが選択されていません | vLLM 互換サービスによって公開されるモデルを更新し、それに Prettify リクエストを送信します。 |

完全な `http` または `https` のベース URL を入力します。 GPT-Voice は、設定を保存するときに周囲の空白と末尾のスラッシュを削除します。 HTTP(S) ではない URL、またはユーザー名またはパスワードを含む URL は拒否されます。 HTTP は、`127.0.0.1`、`localhost`、または `::1` などのループバック エンドポイントでのみ許可されます。すべての非ループバック エンドポイントは HTTPS を使用する必要があります。

アクティブで有効なベース URL がループバックではなくリモートである場合、設定ではプライバシーに関する通知が表示されます。そのエンドポイントを使用すると、選択したテキストと設定した Prettify プロンプトが選択したサービスに送信されます。リモート処理を使用する前に、そのサービスのプライバシー、保持、およびアクセス制御を確認してください。

### vLLM API キー

**vLLM API key** フィールドは、vLLM が選択されている場合にのみ表示されます。 vLLM サービスでベアラー認証が必要な場合に使用します。 GPT-Voice は、キーが設定されている場合にのみ、vLLM リクエストでキーを送信します。

キーは Electron safe storage で個別に保存されます。保存後は、フィールドに再び表示されることはありません。代わりに、キーが保存されていることを示します。フィールドを空白のままにすると、保存されている既存のキーが保持されます。 **Clear API key** を選択して削除します。システムで安全なストレージが利用できない場合、GPT-Voice は新しい vLLM キーを保存できません。

ベース URL、スクリーンショット、またはサポート リクエストにキーを含めないでください。

## モデルの選択と管理アクティブなプロバイダー サービスを開始するか、そのベース URL を変更した後、**Refresh models** を選択します。モデルのリストはアクティブなプロバイダーから取得されるため、プロバイダーを変更した後は再度更新してください。 Prettify を実行する前に、返されたモデルの 1 つを選択します。接続、サービス、認証、またはプロバイダーの応答が失敗した場合、設定には接続エラーまたはモデル更新エラーが表示されます。 URL が有効であること、サービスが実行中であること、および vLLM キーが適切であることを確認してから、再度更新してください。

モデルを選択すると、Ollama には追加の **Model actions** メニューが表示されます。

- **Load model**は、Ollamaに、選択したモデルをGPT-Voiceにロードしたままにするよう要求します。 GPT-Voice が別の Ollama モデルをロードし続けていた場合、最初にそのモデルを解放します。
- **Free model**は、Ollamaに、選択したモデルをメモリから解放するよう要求します。

これらのアクションは Ollama でのみ使用でき、vLLM では使用できません。 GPT-Voice が実行モデルの状態をチェックした後、選択した Ollama モデルには **Loaded** または **Not loaded** が表示されます。 Ollama がサイズを報告すると、設定にはおおよそのモデルまたはロードされた VRAM サイズも表示されます。この値は、GPT-Voice によって保証されるメモリ予約としてではなく、Ollama によって報告される推定値として扱います。

エンドポイントが利用できない場合、またはプロバイダーが要求を拒否した場合、読み込み、解放、および更新が失敗する可能性があります。 [設定] では、現在のフォームを開いたままにして結果またはエラーが表示されるため、エンドポイントまたはモデルを修正して再試行できます。

## テキストの生成方法を制御する

**Temperature** はプライマリ世代制御です。デフォルトは **0** で、許容範囲は **0 to 1** から **0.05** 刻みです。値が低いほど、プロバイダーに要求されるバリエーションが少なくなります。これを変更すると、設定を保存した後に次の Prettify リクエストが変更されます。

**Advanced generation** を開いて残りのコントロールを変更します。折りたたまれた概要には、すべての詳細な値がまだデフォルトを使用しているか、または変更された値がいくつあるかが示されます。これらの設定は、各 Prettify リクエストとともに選択したプロバイダーに送信されます。

|コントロール |デフォルト |受け入れられる値 | |に使用します。
| ------------------------- | ------: | ----------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Top P** | `0.9` | `0.05` – `1`、`0.05` の手順 |選択肢を最も可能性の高い累積確率範囲に制限します。 || **Min P** | `0` | `0` – `1`、`0.05` の手順 |選択したしきい値を下回る確率の低い選択肢を除外します。 |
| **Repeat penalty** | `1` | `0.8` – `1.5`、`0.05` の手順 |プロバイダーが出力の繰り返しを阻止する程度を調整します。 |
| **Top K** | `40` | `1` から `200` までの整数 |各選択肢を最も可能性の高い候補に限定します。 |
| **Maximum output tokens** | `4096` | `1` から `8192` までの整数 |生成される応答の長さに上限を設けます。応答はさらに短くなる可能性があります。 |
| **Seed** | 設定を解除 |空白、または `0` から `2147483647` までの整数 |オプションの数値シードをプロバイダーに提供します。これはリクエストの再現に役立ちますが、結果はモデルやサービスのバージョンによって異なる場合があります。 |

10 進数の制御では 0.05 増分が使用されます。トップ P は 0.05 ～ 1 を受け入れます。 Min P は 0 ～ 1 を受け入れます。繰り返しペナルティは 0.8 ～ 1.5 を受け入れます。 Top K は 1 ～ 200 の整数を受け入れますが、最大出力トークンは 1 ～ 8192 の整数を受け入れます。

選択したモデルとサービスの要件がわからない場合は、デフォルトを使用してください。 GPT-Voice は、同等の生成の選択肢を Ollama と vLLM に送信しますが、どちらのサービスでも、リクエストを拒否したり、独自のモデル サポートに従って設定を処理したりできます。

## Prettify プロンプトを作成する

**Prompt** は必須で、デフォルトは GPT-Voice の組み込みの保守的なコピーエディタ命令です。これは、選択したテキストを不活性なソース素材として扱い、その言語と意味を保持し、修正して明確にし、不要な繰り返しを削除し、編集されたテキストのみを返すようにサービスに指示します。また、選択したテキストに含まれる命令を実行しないようサービスに指示します。

プロンプトを別の編集ポリシーに置き換えることができます。 **4,000 characters or fewer** のままにしておきます。空のプロンプト、4,000 文字を超えるプロンプト、サポートされていないプロバイダー、無効なエンドポイント、空のモデル、または範囲外の生成値はブロック **Save changes** をブロックし、影響を受けるフィールドを特定します。設定を保存すると、先頭と末尾の空白が削除されます。

プロンプトは、選択したテキストとともにアクティブなプロバイダーに送信されます。プロバイダーに受け取られたくないパスワード、API キー、個人データ、または機密指示は含めないでください。

## 変更を保存して検証するプロバイダー、モデル、プロンプト、および生成の値は、設定フォームの一部です。編集により **Unsaved changes** が作成されます。フォームに検証エラーがない場合にのみ、**Save changes** を選択してください。保存が成功すると、通常の Prettify 構成が保持され、設定が閉じられます。 vLLM キーは別個のままです。キーは Electron safe storage を通じてのみ保存され、設定ビューに返されることはありません。

保存に失敗した場合、[設定] ウィンドウは開いたままになり、安全なエラー メッセージが表示されます。報告されたフィールドまたはプロバイダー接続を修正して、再試行してください。未保存変更および破棄確認の動作については、[設定の概要](index.md)を参照してください。

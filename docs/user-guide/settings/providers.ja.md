# プロバイダー設定

GPT-Voice Command Dock で文字起こしプロバイダーを選択します。現在選択されているプロバイダーを構成するには、ヘッダー内の **Connect** または **Configure** コントロールを開きます。プロバイダー ダイアログには、そのプロバイダーに適用されるコントロールのみが表示されます。

## ChatGPT Web

ChatGPT Web は API キーではなくブラウザ セッションを使用します。プロバイダーのダイアログには、セッションが保存されているかどうかが表示されます。

1. Command Dockで**ChatGPT Web**を選択します。
2. プロバイダダイアログを開き、セッションが保存されていない場合は **Log in** を選択するか、保存されたセッションを置き換えるには **Log in again** を選択します。
3. GPT-Voice ブラウザ ウィンドウでサインインを完了し、プロバイダーに **Connected** が表示されたら Command Dock に戻ります。

GPT-Voice が保存した ChatGPT Web 認証を忘れたい場合は、**Clear session** を使用します。 GPT-Voice は、クリアする前に確認を求めます。セッションをクリアすると、このプロバイダーが切断されます。文字起こしに使用する前に、再度サインインしてください。

アカウント所有権、ブラウザーセッション、プロバイダー制限の詳細については、[プロバイダー](../guides/providers.md)を参照してください。

## OpenAI API

**OpenAI API** を選択し、プロバイダー ダイアログを開いてこれらのフィールドを設定します。独自の OpenAI API キーとアカウントを使用してください。 GPT-Voice は、キー、クレジット、または OpenAI サービスへのアクセスを提供しません。

| フィールド      | 現在の動作                                                                                                                                             |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **API key**     | 新しいキーを入力して保存します。キーが保存されている場合でも、再度開くとフィールドは空白になり、空に保存してもその保存されたキーは置き換えられません。 |
| **Model**       | `whisper-1` は、利用可能な唯一の転写モデルです。これは読み取り専用フィールドとして表示されます。                                                       |
| **Language**    | 自動検出 (デフォルト)、英語、ロシア語、ウクライナ語、またはベラルーシ語を選択します。                                                                  |
| **Prompt**      | オプションの文字起こし指導。デフォルトは空です。先頭と末尾の空白は保存時に削除されます。                                                               |
| **Temperature** | 転写のバリエーションを 0 から 1 まで制御します。デフォルトは 0 です。コントロールは0.05ステップで変化します。                                          |

**Save** を選択して変更を検証し、保存します。保存が成功すると、ダイアログが閉じます。無効なモデル、言語、または温度の値は拒否されます。保存に失敗した場合、ダイアログには安全なエラー メッセージが表示され、開いたままになります。

## 保存された資格情報と認証のクリア

GPT-Voice は、Electron safe storage を通じてのみ OpenAI API キーを保存します。キー自体はダイアログに表示されません。ダイアログには、代わりに API キーが保存されていることが示されます。安全なストレージが利用できない場合、GPT-Voice は新しいキーを保存できません。**Clear API key** を使用してダイアログを確認し、他の OpenAI API 設定を維持したまま保存されたキーを削除します。このボタンはキーが保存されている場合にのみ使用できます。 OpenAI API が再度転写される前に、新しいキーを入力して保存する必要があります。

プロバイダーの資格情報とサービスの使用状況は、プロバイダー アカウントの条件、請求、割り当て、およびプライバシー ポリシーに基づいて維持されます。サポート リクエスト、スクリーンショット、またはドキュメントにキーを貼り付けないでください。

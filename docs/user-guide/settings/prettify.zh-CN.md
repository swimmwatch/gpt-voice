# 美化设置

**Prettify** 是 GPT-Voice 对选定文本的意义保留清理操作。打开 **Settings** 并选择 **Prettify** 以选择服务、其模型及其生成行为。有关选择文本并开始操作的信息，请参阅[翻译和美化所选文本](../guides/text-actions.md)。

Prettify 需要您运行的 Ollama 或 vLLM 服务，以及为该服务选择的型号。 GPT-Voice 不会下载、启动、托管或支付任何一项服务的费用。型号字段为必填项；如果未配置模型，Prettify 会报告需要模型，而不是发送选定的文本。

## 选择一个提供商并连接它

在**Provider**中选择**Ollama**或**vLLM**。 GPT-Voice 为每个提供商单独保留基本 URL 和所选模型，因此切换提供商不会替换其他提供商的选择。默认提供程序是 Ollama。

| 供应商     | 默认基本 URL               | 型号默认     | 连接行为                                                 |
| ---------- | -------------------------- | ------------ | -------------------------------------------------------- |
| **Ollama** | `http://127.0.0.1:11434`   | 没有选择型号 | 刷新 Ollama 服务中可用的模型并向其发送 Prettify 请求。   |
| **vLLM**   | `http://127.0.0.1:8000/v1` | 没有选择型号 | 刷新由 vLLM 兼容服务公开的模型并向其发送 Prettify 请求。 |

输入完整的 `http` 或 `https` 基本 URL。 GPT-Voice 保存设置时会删除周围的空格和尾部斜杠。非 HTTP(S) 或包含用户名或密码的 URL 将被拒绝。仅允许环回端点使用 HTTP，例如 `127.0.0.1`、`localhost` 或 `::1`；每个非环回端点必须使用 HTTPS。

当活动的有效基本 URL 是远程而不是环回时，“设置”会显示隐私声明。使用该端点将选定的文本和您配置的美化提示发送到您选择的服务。在使用远程处理之前，请检查该服务的隐私、保留和访问控制。

### vLLM API 密钥

仅当选择 vLLM 时，才会出现 **vLLM API key** 字段。当您的 vLLM 服务需要承载身份验证时使用它。仅当配置了密钥时，GPT-Voice 才会通过 vLLM 请求发送密钥。

该密钥与Electron safe storage单独存储。保存后，该字段不再显示；相反，它表示已存储密钥。将该字段留空会保留现有的存储密钥。选择 **Clear API key** 将其删除。如果您的系统上没有安全存储，GPT-Voice 无法保存新的 vLLM 密钥。

请勿将密钥放入基本 URL、屏幕截图或支持请求中。

## 选择并管理模型启动活动提供商服务或更改其基本 URL 后选择 **Refresh models**。模型列表来自活动的提供商，因此更改提供商后请再次刷新。在运行 Pretify 之前选择返回的模型之一。如果连接、服务、身份验证或提供程序响应失败，“设置”将显示连接或模型刷新错误；检查 URL 是否有效、服务是否正在运行以及 vLLM 键是否正确，然后再次刷新。

选择模型时，Ollama 显示附加的 **Model actions** 菜单：

- **Load model** 要求 Ollama 保持为 GPT-Voice 加载所选模型。如果 GPT-Voice 已加载不同的 Ollama 模型，它会首先释放该模型。
- **Free model** 要求 Ollama 从内存中释放所选模型。

这些操作仅适用于 Ollama，不适用于 vLLM。在 GPT-Voice 检查其运行模型状态后，选定的 Ollama 模型显示 **Loaded** 或 **Not loaded**。当 Ollama 报告大小时，“设置”还会显示近似模型或加载的 VRAM 大小。将该值视为 Ollama 报告的估计值，而不是 GPT-Voice 保证的内存预留。

如果端点不可用或提供者拒绝请求，则加载、释放和刷新可能会失败。设置使当前表单保持打开状态并显示结果或错误，以便您可以更正端点或模型并重试。

## 控制文本的生成方式

**Temperature** 是主要的生成控制。其默认值为 **0**，允许范围为 **0 to 1**，步长为 **0.05**。较低的值要求提供者提供较小的变化；更改它会更改保存设置后的下一个美化请求。

打开 **Advanced generation** 更改其余控件。折叠的摘要指示所有高级值是否仍使用其默认值或有多少已更改。这些设置随每个 Prettify 请求一起发送到所选提供商。

| 控制                      |     默认 | 接受值                                  | 用它来                                                                             |
| ------------------------- | -------: | --------------------------------------- | ---------------------------------------------------------------------------------- |
| **Top P**                 |    `0.9` | `0.05`–`1`，以 `0.05` 步                | 将选择限制在最可能的累积概率范围内。                                               |     | **Min P** | `0` | `0`–`1`，以 `0.05` 步 | 排除低于所选阈值的较低概率选择。 |
| **Repeat penalty**        |      `1` | `0.8`–`1.5`，以 `0.05` 步               | 调整提供者阻止重复输出的强度。                                                     |
| **Top K**                 |     `40` | 从 `1` 到 `200` 的整数                  | 将每个选择限制为最有可能的候选人。                                                 |
| **Maximum output tokens** |   `4096` | 从 `1` 到 `8192` 的整数                 | 限制生成的响应的长度。响应还可以更短。                                             |
| **Seed**                  | 取消设置 | 空白或从 `0` 到 `2147483647` 之间的整数 | 向提供者提供可选的数字种子。它可以帮助重现请求，但结果仍可能因模型和服务版本而异。 |

小数控件使用 0.05 增量。 Top P 接受 0.05 到 1； Min P 接受 0 到 1；重复罚分接受 0.8 至 1.5。 Top K 接受 1 到 200 之间的整数，而最大输出令牌接受 1 到 8192 之间的整数。

除非您知道所选模型和服务的要求，否则请使用默认值。 GPT-Voice 向 Ollama 和 vLLM 发送等效的生成选择，但任一服务仍然可以拒绝请求或根据其自己的模型支持处理设置。

## 编写美化提示

**Prompt** 是必需的，默认为 GPT-Voice 的内置保守复制编辑器指令。它告诉服务将选定的文本视为惰性源材料，保留其语言和含义，更正和澄清它，删除不必要的重复，并仅返回编辑后的文本。它还指示服务不要执行所选文本中包含的指令。

您可以将提示替换为不同的编辑策略。将其保留为 **4,000 characters or fewer**。空白提示、超过 4,000 个字符的提示、不受支持的提供程序、无效端点、空模型或超出范围的生成值会阻止 **Save changes** 并标识受影响的字段。保存设置时，前导和尾随空格将被删除。

提示将与选定的文本一起发送给活动提供者。请勿包含您不希望提供商收到的密码、API 密钥、个人数据或机密说明。

## 保存并验证更改提供程序、模型、提示和生成值是“设置”表单的一部分。编辑创建 **Unsaved changes**；仅在表单没有验证错误后才选择 **Save changes**。成功保存会保留常规 Prettify 配置并关闭“设置”。 vLLM 键保持独立：它仅通过 Electron safe storage 存储，并且永远不会在设置视图中返回。

如果保存失败，“设置”窗口将保持打开状态，并显示安全错误消息。更正报告的字段或提供商连接，然后重试。有关未保存更改和放弃确认行为，请参阅[设置概述](index.md)。

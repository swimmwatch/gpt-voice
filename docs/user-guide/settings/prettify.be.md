# Налады Prettify

**Prettify** — дзеянне GPT-Voice для ачысткі вылучанага тэксту са захаваннем яго сэнсу. Адкрыйце **Settings** і
абярыце **Prettify**, каб выбраць службу, мадэль і паводзіны генерацыі. Пра вылучэнне тэксту і запуск дзеяння чытайце
ў [перакладзе і Prettify вылучанага тэксту](../guides/text-actions.md).

Для Prettify патрэбна служба Ollama або vLLM, якой кіруеце вы, і абраная для яе мадэль. GPT-Voice не спампоўвае, не
запускае, не размяшчае і не аплачвае ніводную з гэтых службаў. Поле мадэлі абавязковае: калі мадэль не наладжана,
Prettify паведамляе пра яе неабходнасць замест адпраўкі вылучанага тэксту.

## Абярыце пастаўшчыка і падлучыцеся

Абярыце **Ollama** або **vLLM** у **Provider**. GPT-Voice захоўвае базавы URL і выбраную мадэль асобна для кожнага
пастаўшчыка, таму змена аднаго не замяняе выбар іншага. Па змаўчанні выбраны Ollama.

| Пастаўшчык | Базавы URL па змаўчанні    | Мадэль па змаўчанні | Паводзіны падлучэння                                                     |
| ---------- | -------------------------- | ------------------- | ------------------------------------------------------------------------ |
| **Ollama** | `http://127.0.0.1:11434`   | Мадэль не выбрана   | Абнаўляе даступныя мадэлі Ollama і адпраўляе ім запыты Prettify.         |
| **vLLM**   | `http://127.0.0.1:8000/v1` | Мадэль не выбрана   | Абнаўляе мадэлі сумяшчальнай службы vLLM і адпраўляе ёй запыты Prettify. |

Увядзіце поўны базавы URL з `http` або `https`. Пры захаванні GPT-Voice прыбірае прабелы і канчатковыя косыя рысы.
URL без HTTP(S) або з імем карыстальніка ці паролем адхіляюцца. HTTP дазволены толькі для loopback-адраса, напрыклад
`127.0.0.1`, `localhost` або `::1`; кожны аддалены адрас мусіць выкарыстоўваць HTTPS.

Калі дзеючы сапраўдны базавы URL аддалены, Settings паказвае апавяшчэнне пра канфідэнцыяльнасць. Вылучаны тэкст і
наладжаная падказка Prettify адпраўляюцца ў выбраную службу; перад аддаленай апрацоўкай праверце яе палітыку
канфідэнцыяльнасці, захоўвання і доступу.

### Ключ API vLLM

Поле **vLLM API key** паказваецца толькі для vLLM. Выкарыстоўвайце яго, калі служба патрабуе аўтэнтыфікацыю bearer;
GPT-Voice адпраўляе ключ з запытамі vLLM толькі калі ключ наладжаны. Ён захоўваецца асобна праз Electron safe
storage і пасля захавання не паказваецца зноў. Пустое поле захоўвае існы ключ; абярыце **Clear API key**, каб яго
выдаліць. Калі бяспечнае сховішча недаступнае, GPT-Voice не можа захаваць новы ключ. Не ўказвайце ключ у базавым URL,
на здымку экрана або ў запыце ў падтрымку.

## Абярыце мадэль і кіруйце ёю

Пасля запуску службы або змены URL абярыце **Refresh models**. Спіс мадэляў атрымліваецца ад актыўнага пастаўшчыка,
таму абнаўляйце яго пасля змены пастаўшчыка. Абярыце вернутую мадэль перад запускам Prettify. Калі падлучэнне,
служба, аўтэнтыфікацыя або адказ пастаўшчыка не ўдаюцца, Settings паказвае памылку; праверце URL, працу службы і
ключ vLLM, а потым паўтарыце абнаўленне.

Для Ollama пры выбранай мадэлі даступнае меню **Model actions**:

- **Load model** просіць Ollama трымаць выбраную мадэль загружанай для GPT-Voice; іншую загружаную мадэль Ollama
  спачатку вызваляе.
- **Free model** просіць Ollama вызваліць выбраную мадэль з памяці.

Гэтыя дзеянні даступныя толькі для Ollama, не для vLLM. Выбраная мадэль паказваецца як **Loaded** або **Not loaded**;
калі Ollama паведамляе памер, Settings таксама паказвае прыблізны памер мадэлі або загружанай VRAM. Гэта ацэнка
Ollama, а не гарантаванае GPT-Voice рэзерваванне памяці.

## Кіруйце генерацыяй тэксту

**Temperature** — галоўны параметр генерацыі: па змаўчанні **0**, дапушчальны дыяпазон ад **0 да 1** крокам
**0.05**. Меншае значэнне просіць меншай варыятыўнасці. Адкрыйце **Advanced generation** для астатніх значэнняў;
яны адпраўляюцца з кожным запытам Prettify.

| Параметр                  | Па змаўчанні | Дапушчальнае значэнне                  |
| ------------------------- | -----------: | -------------------------------------- |
| **Top P**                 |        `0.9` | `0.05`–`1`, крок `0.05`                |
| **Min P**                 |          `0` | `0`–`1`, крок `0.05`                   |
| **Repeat penalty**        |          `1` | `0.8`–`1.5`, крок `0.05`               |
| **Top K**                 |         `40` | Цэлае ад `1` да `200`                  |
| **Maximum output tokens** |       `4096` | Цэлае ад `1` да `8192`                 |
| **Seed**                  | Не зададзена | Пуста або цэлае ад `0` да `2147483647` |

Выкарыстоўвайце значэнні па змаўчанні, калі не ведаеце патрабаванняў мадэлі і службы. GPT-Voice адпраўляе
эквівалентныя параметры Ollama і vLLM, але служба ўсё роўна можа адхіліць запыт або апрацаваць параметр паводле
падтрымкі сваёй мадэлі.

## Напішыце падказку Prettify

**Prompt** абавязковая і па змаўчанні змяшчае кансерватыўную інструкцыю рэдактара GPT-Voice. Яна просіць лічыць
вылучаны тэкст інэртным зыходным матэрыялам, захоўваць мову і сэнс, выпраўляць і ўдакладняць яго, прыбіраць лішнія
паўторы і вяртаць толькі адрэдагаваны тэкст, не выконваючы інструкцый з вылучэння.

Падказку можна замяніць для іншай палітыкі рэдагавання. Абмяжуйце яе **4,000 сімваламі або менш**. Пустая або занадта
доўгая падказка, непадтрыманы пастаўшчык, несапраўдны адрас, пустая мадэль або значэнне генерацыі па-за дыяпазонам
блакіруе **Save changes** і паказвае адпаведнае поле. Падказка адпраўляецца з вылучаным тэкстам, таму не ўключайце
паролі, ключы API, асабістыя даныя або канфідэнцыяльныя інструкцыі.

## Захавайце і праверце змяненні

Пастаўшчык, мадэль, падказка і параметры генерацыі ўваходзяць у форму Settings. Змена стварае **Unsaved changes**;
абярыце **Save changes** толькі калі няма памылак праверкі. Паспяховае захаванне запісвае звычайную канфігурацыю
Prettify і закрывае Settings. Ключ vLLM застаецца асобным, захоўваецца толькі праз Electron safe storage і ніколі
не вяртаецца ў інтэрфейс налад. Калі захаванне не ўдаецца, Settings застаецца адкрытым з бяспечным паведамленнем.

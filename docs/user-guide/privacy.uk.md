# Конфіденційність і дані

GPT-Voice обробляє мовлення, виділений текст, облікові дані та налаштування браузера. На цій сторінці пояснюється поточний шлях до даних
і елементи керування, доступні для видалення даних. Це не замінює політику конфіденційності чи умови будь-якої служби, яку ви виберете
використовувати.

## Потоки даних

GPT-Voice надсилає дані за межі вашого комп’ютера, лише якщо ви використовуєте функцію, що підтримується зовнішньою службою:

| Особливість                    | Дані надіслані                                           | Пункт призначення                                       |
| ------------------------------ | -------------------------------------------------------- | ------------------------------------------------------- |
| Транскрипція з **ChatGPT Web** | Підготовлений запис                                      | ChatGPT через сеанс веб-переглядача, у якому ви ввійшли |
| Транскрипція з **OpenAI API**  | Підготовлений запис і налаштовані параметри транскрипції | Кінцева точка аудіотранскрипції OpenAI                  |
| **Translate**                  | Виділений текст                                          | Google Translate                                        |
| **Prettify**                   | Виділений текст і ваш налаштований запит Prettify        | Ваша налаштована Ollama або vLLM кінцева точка          |

Використовуйте облікові записи та кінцеві точки, яким ви довіряєте, і перегляньте умови обробки даних у них. Локальна кінцева точка Ollama або vLLM
зберігає запит на машині, на якій виконується ця служба; віддалена кінцева точка отримує текст. Використовуйте HTTPS для a
кінцева точка Prettify без петлі. Сервіси на основі браузера можуть використовувати проксі-сервер, налаштований у [Налаштуваннях мережі](settings/network.md).

GPT-Voice записує успішні результати транскрипції, перекладу та Prettify у системний буфер обміну. Операційний
система та інші програми з доступом до буфера обміну можуть зберігати або читати це значення; очистіть або замініть його після вставлення
чутливий вихід.

## Локальні дані та тимчасова пам'ять

| Дані                                   | Де і як довго зберігається                                                                                                                                            |
| -------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Історія успішної транскрипції          | Локальні дані SQLite за адресою `gpt-voice.sqlite3` із часом запиту, ідентифікатором і назвою провайдера та текстом транскрипції. Записаний звук не зберігається.     |
| Повторний запис                        | Останній підготовлений аудіо зберігається лише в пам’яті запущеної програми після невдалої транскрипції. Початок нового запису або перезапуск GPT-Voice видаляє його. |     | Упорядкування кешу результатів | До 20 результатів зберігаються в пам'яті до 60 секунд. Контекст кешу залежить від вибраного тексту, підказки та налаштованих параметрів постачальника; він видаляється, коли GPT-Voice виходить. |
| Налаштування та ідентифікація браузера | Зберігається в даних локальної програми GPT-Voice і використовується для наступних запусків.                                                                          |
| ChatGPT Web автентифікація             | Локальний сеанс браузера та дані автентифікації. Використовуйте елемент керування постачальника, щоб очистити його.                                                   |

Дії з виділеним текстом тимчасово читають і замінюють вміст буфера обміну, поки вони збирають виділення. Якщо Переклад або
Prettify не вдається або скасовується, GPT-Voice відновлює значення буфера обміну, яке воно зафіксувало до дії. На успіх,
результат залишається в буфері обміну, щоб ви могли його вставити.

## Облікові дані та вимоги до шифрування

Ключі OpenAI API, vLLM ключі API та паролі проксі HTTP/HTTPS зберігаються через Electron safe storage, коли це
захист доступний. GPT-Voice не повертає ці збережені значення до перегляду налаштувань. Якщо безпечне зберігання є
недоступний, GPT-Voice не може зберегти новий секрет через цей елемент керування.

Це не загальне твердження про шифрування для кожного файлу в каталозі даних програми. Зокрема, транскрипція
історія, звичайні налаштування та дані сеансу ChatGPT Web мають власну поведінку локального зберігання. Не передавайте ключі API,
паролі проксі, інформацію про сеанс, продиктований текст або знімки екрана, які їх містять.

## Видалити або скинути дані

Виберіть найвужчий елемент керування, який відповідає вашим потребам:

1. У вікні «Історія» використовуйте **Clear history**, щоб остаточно видалити всі збережені записи транскрипції. див
   [історія та трей](guides/history-and-tray.md).
2. В елементах керування постачальником транскрипції використовуйте **Clear session** для ChatGPT Web або **Clear API key** для OpenAI API.
   Перегляньте [налаштування постачальника](settings/providers.md).
3. Використовуйте елемент керування **Clear API key** у [Налаштування покращення](settings/prettify.md) та елемент керування паролем **Clear**
   у [Налаштуваннях мережі](settings/network.md), якщо це можливо.
4. Замініть або очистіть системний буфер обміну окремо, якщо він містить вихідні дані, які більше не потрібно вставляти.
5. Для повного локального скидання закрийте GPT-Voice із трея та видаліть збережений каталог даних програми:
   `%APPDATA%\GPT-Voice` у Windows або `~/.config/GPT-Voice` у Linux. Це видаляє локальні програми
   налаштувань, історії та збережених даних провайдера, і потребує повторного налаштування після перевстановлення або повторного запуску.

Видалення локальних даних необоротне. Видалення самої програми навмисно зберігає ці каталоги; побачити
[встановіть, оновіть або видаліть](install.md) для поведінки видалення для певної платформи.

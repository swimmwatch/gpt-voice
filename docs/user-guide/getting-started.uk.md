# Перше використання: підключіть постачальника та розшифруйте мовлення

Після [встановлення GPT-Voice](install.md) підключіть одного постачальника розшифрування, надайте доступ до
мікрофона й зробіть короткий запис. GPT-Voice копіює успішну розшифровку до буфера обміну; він не вводить її
автоматично в програму, якою ви користувалися.

## 1. Виберіть постачальника розшифрування

Відкрийте GPT-Voice і виберіть **ChatGPT Web** або **OpenAI API** у селекторі **Provider** в Command Dock.

| Постачальник    | Що вам потрібно                                           | Дія під час першого використання                                             |
| --------------- | --------------------------------------------------------- | ---------------------------------------------------------------------------- |
| **ChatGPT Web** | Обліковий запис ChatGPT, до якого можна ввійти.           | Виберіть його, потім **Connect** і завершіть вхід у браузері.                |
| **OpenAI API**  | Власний ключ OpenAI API та доступна оплата або квота API. | Виберіть його, потім **Configure**, щоб відкрити налаштування постачальника. |

Доступність постачальника, ліміти, оплата й умови залежать від облікового запису, яким ви користуєтеся. GPT-Voice
не обходить ці обмеження.

### Підключіть ChatGPT Web

1. Виберіть **ChatGPT Web**.
2. Виберіть **Connect**.
3. Завершіть вхід у ChatGPT у вікні браузера, що відкриється.
4. Коли ChatGPT буде готовий, закрийте це вікно входу й поверніться до GPT-Voice.

Коли сеанс буде готовий, стан постачальника зміниться на **Connected**. GPT-Voice зберігає сеанс браузера у ваших
локальних даних програми й автоматично запускає фоновий браузер під час наступних запусків. Якщо термін дії сеансу
спливе, знову виберіть **Connect**, щоб повторно ввійти.

### Налаштуйте OpenAI API

1. Виберіть **OpenAI API**.
2. Виберіть **Configure**. Також можна скористатися елементом налаштувань постачальника поруч із селектором.
3. Вставте свій ключ OpenAI API.
4. За потреби виберіть мову розшифрування, запит або температуру.
5. Виберіть **Save**.

Модель розшифрування фіксована: `whisper-1`. Після успішного збереження GPT-Voice показує постачальника як
**Connected** і повідомляє, що його налаштовано. Програма зберігає ключ API локально через Electron safe storage,
якщо воно доступне; ключ не відображається знову в інтерфейсі. Розшифрування OpenAI API не використовує браузер.

## 2. Надайте доступ до мікрофона

Перший запис запитує в операційної системи дозвіл на використання мікрофона. Надайте доступ GPT-Voice, а потім
поверніться до Command Dock. Якщо в доступі відмовлено або мікрофон недоступний, стан покаже **Error: Could not
access microphone** і аудіо не надсилатиметься. Увімкніть дозвіл у налаштуваннях конфіденційності операційної системи
перед наступною спробою.

## 3. Зробіть перший запис

1. Виберіть постачальника зі станом **Connected**.
2. Виберіть **Start recording** або натисніть показане сполучення клавіш для запису (типове значення — `F9`).
3. Скажіть коротке речення. Стан зміниться на **Recording**.
4. Виберіть **Stop recording** (типове значення — `F10`). GPT-Voice змінить стан на **Transcribing**, коли надсилатиме
   захоплене аудіо вибраному постачальнику.
5. Дочекайтеся **Copied to clipboard**, а потім вставте текст у будь-яке текстове поле, щоб підтвердити результат.

Також можна призупинити, відновити або скасувати активний запис із Command Dock. Скасування відкидає активний запис
замість надсилання його на розшифрування.

## Якщо перша розшифровка не працює

- Якщо **Connected** не показується, знову відкрийте елементи керування постачальником і завершіть вхід або збережіть
  дійсний ключ API. Для ChatGPT Web стан натомість може повідомляти про помилку ініціалізації браузера або завершення
  сеансу.
- Якщо стан повідомляє про помилку мікрофона, надайте GPT-Voice доступ до мікрофона в операційній системі й зробіть
  новий запис.
- Якщо стан показує **Transcription failed** або **Transcription error**, перевірте обліковий запис, підключення та
  ліміти вибраного постачальника, а потім повторіть спробу. Невдала розшифровка не копіюється до буфера обміну.

Наступні сторінки посібника докладніше описують елементи керування записом, поведінку постачальника, сполучення
клавіш і усунення несправностей. Поки що постачальник зі станом **Connected** і результат **Copied to clipboard**
підтверджують, що основний шлях працює.

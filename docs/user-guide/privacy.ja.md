## データフロー

GPT-Voice は、外部サービスによる機能を使用する場合にのみ、コンピューターの外部にデータを送信します。

| 特集                       | 送信されたデータ                               | 目的地                                              |
| -------------------------- | ---------------------------------------------- | --------------------------------------------------- |
| **ChatGPT Web** による転写 | 準備された録音                                 | サインインしたブラウザー セッションを通じて ChatGPT |
| **OpenAI API** による転写  | 準備された録音と設定された文字起こしオプション | OpenAI の音声文字起こしエンドポイント               |
| **Translate**              | 選択したテキスト                               | Google Translate                                    |
| **Prettify**               | 選択したテキストと設定した Prettify プロンプト | 構成済みの Ollama または vLLM エンドポイント        |

信頼できるアカウントとエンドポイントを使用し、データ処理条件を確認してください。ローカル ループバック Ollama または vLLM エンドポイント
そのサービスを実行しているマシン上でリクエストを保持します。リモート エンドポイントがテキストを受信します。 HTTPS を使用する
非ループバック Prettify エンドポイント。ブラウザベースのサービスは、[ネットワーク設定](settings/network.md)で設定されたプロキシを使用できます。

GPT-Voice は、成功した転写、翻訳、Prettify の結果をシステム クリップボードに書き込みます。動作
クリップボードにアクセスできるシステムおよびその他のアプリケーションは、その値を保持または読み取ることができます。貼り付け後はクリアするか交換してください
敏感な出力。

## ローカルデータと一時メモリ

| データ                 | どこに、どのくらいの期間保管されますか                                                                                                                                         |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 成功した文字起こし履歴 | `gpt-voice.sqlite3` のローカル SQLite データ (リクエスト時刻、プロバイダー ID と名前、転写テキストを含む)。録音された音声は保存されません。                                    |
| 再試行可能な記録       | 最も新しく準備された音声は、文字起こしが失敗した後、実行中のアプリケーションのメモリにのみ保持されます。新しい録画を開始するか、GPT-Voice を再起動すると、それが削除されます。 |     | 結果キャッシュをきれいにする | 最大 20 件の結果が最大 60 秒間メモリに保持されます。キャッシュ コンテキストは、選択したテキスト、プロンプト、および構成されたプロバイダー設定によって異なります。 GPT-Voice が終了すると削除されます。 |
| 設定とブラウザの ID    | GPT-Voice のローカル アプリケーション データに保存され、後の起動に使用されます。                                                                                               |
| ChatGPT Web 認証       | ローカルブラウザセッションと認証データ。プロバイダー コントロールを使用してクリアします。                                                                                      |

選択テキスト アクションは、選択範囲を収集するときに一時的にクリップボードの内容を読み取り、置き換えます。翻訳または
Prettify が失敗するかキャンセルされた場合、GPT-Voice はアクション前にキャプチャしたクリップボード値を復元します。成功すると、
結果はクリップボードに残り、貼り付けることができます。

## 認証情報と暗号化の資格

OpenAI API キー、vLLM API キー、および HTTP/HTTPS プロキシ パスワードは、Electron safe storage を通じて保存されます。
保護が利用可能です。 GPT-Voice は、保存された値を設定ビューに返しません。安全なストレージがあれば、
利用できないため、GPT-Voice はそのコントロールを通じて新しいシークレットを保存できません。

これは、アプリケーション データ ディレクトリ内のすべてのファイルに対する包括的な暗号化要求ではありません。特に転写
履歴、通常の設定、および ChatGPT Web セッション データには、独自のローカル ストレージ動作があります。 API キーを共有しないでください。
プロキシ パスワード、セッション情報、口述テキスト、またはそれらを含むスクリーンショット。

## データを削除またはリセットする

ニーズを満たす最も狭いコントロールを選択してください。

1. [履歴] ウィンドウで **Clear history** を使用して、保存されている転写エントリをすべて完全に削除します。参照
   [歴史とトレイ](guides/history-and-tray.md)。
2. 文字起こしプロバイダー コントロールで、ChatGPT Web の場合は **Clear session** を、OpenAI API の場合は **Clear API key** を使用します。
   [プロバイダー設定](settings/providers.md)を参照してください。
3. [Prettify 設定](settings/prettify.md) の **Clear API key** コントロールとパスワード **Clear** コントロールを使用します。
   該当する場合は、[ネットワーク設定](settings/network.md)]で。
4. 貼り付けたくない出力がシステム クリップボードに含まれている場合は、システム クリップボードを個別に交換またはクリアします。
5. 完全なローカル リセットの場合は、トレイから GPT-Voice を終了し、保持されているアプリケーション データ ディレクトリを削除します。
   Windows の場合は `%APPDATA%\GPT-Voice`、Linux の場合は `~/.config/GPT-Voice`。これにより、ローカルのアプリケーション管理が削除されます
   設定、履歴、保存されたプロバイダー データが保存されているため、再インストールまたは再起動後に再度セットアップする必要があります。

ローカル データの削除は元に戻せません。アプリケーションのみをアンインストールすると、これらのディレクトリは意図的に保持されます。参照してください
プラットフォーム固有のアンインストール動作については、[install、update、remove](install.md)。

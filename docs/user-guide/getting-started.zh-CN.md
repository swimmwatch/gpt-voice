# 首次使用：连接提供商并转录语音

[安装 GPT-Voice](install.md) 后，连接一个转录提供程序，允许麦克风访问，并创建一个
简短的录音。 GPT-Voice 将成功的转录复制到剪贴板；它不会自动输入
您正在使用的应用程序。

## 1. 选择转录提供商

打开GPT-Voice并在Command Dock的**Provider**选择器中选择**ChatGPT Web**或**OpenAI API**。

| 供应商          | 您需要什么                                        | 第一次行动                                      |
| --------------- | ------------------------------------------------- | ----------------------------------------------- |
| **ChatGPT Web** | 您可以登录的 ChatGPT 帐户。                       | 选择它，然后选择**Connect**并完成浏览器登录。   |
| **OpenAI API**  | 您自己的 OpenAI API 密钥和可用的 API 计费或配额。 | 选择它，然后选择 **Configure** 打开提供商设置。 |

提供商的可用性、限制、计费和条款由您使用的帐户控制。 GPT-Voice 不绕过
那些限制。

### 连接 ChatGPT Web

1. 选择**ChatGPT Web**。
2. 选择**Connect**。
3. 在打开的浏览器窗口中完成 ChatGPT 登录。
4. ChatGPT 准备就绪后，关闭该登录窗口并返回到 GPT-Voice。

当会话准备就绪时，提供程序状态更改为 **Connected**。 GPT-Voice 将浏览器会话保存在您的
本地应用程序数据并在以后启动时自动启动其后台浏览器。如果会话过期，请选择
再次**Connect**再次登录。

### 配置OpenAI API

1. 选择**OpenAI API**。
2. 选择**Configure**。您还可以使用提供程序选择器旁边的提供程序设置控件。
3. 粘贴您的 OpenAI API 密钥。
4. （可选）选择转录语言、提示或温度。
5. 选择**Save**。

转录模型固定为 `whisper-1`。成功保存后，GPT-Voice 将提供程序显示为
**Connected** 并报告提供程序已配置。应用程序使用 Electron 在本地存储 API 密钥
可用时进行安全存储；该密钥不会显示在界面中。 OpenAI API 转录不使用
浏览器。

## 2. 允许麦克风访问

第一次录音会要求您的操作系统授予麦克风权限。允许访问 GPT-Voice，然后返回
Command Dock。如果访问被拒绝或没有可用的麦克风，状态将显示 **Error: Could not access
麦克风**，并且不发送任何音频。尝试之前，请在操作系统的隐私控制中启用权限
再次。

## 3. 进行第一次录音

1. 选择显示 **Connected** 的提供商。
2. 选择**Start recording**或按显示的录制快捷键（默认为`F9`）。3. 说一个短句。状态更改为 **Recording**。
3. 选择 **Stop recording**（默认 `F10`）。 GPT-Voice 在发送时将状态更改为 **Transcribing**
   捕获音频到选定的提供商。
4. 等待 **Copied to clipboard**，然后粘贴到任意文本字段以确认结果。

您还可以从 Command Dock 暂停、恢复或取消活动录制。取消会丢弃活动的
录制而不是发送进行转录。

## 如果第一个转录不起作用

- 如果未显示 **Connected**，请重新打开提供程序控件并完成登录或保存有效的 API 密钥。对于
  ChatGPT Web，状态可能会显示浏览器初始化失败或会话已过期。
- 如果状态报告麦克风错误，请允许GPT-Voice在操作系统中使用您的麦克风并进行
  新录音。
- 如果状态报告 **Transcription failed** 或 **Transcription error**，请检查所选提供商的帐户，
  连接和限制，然后重试。失败的转录不会复制到剪贴板。

接下来的指南页面更详细地介绍了录制控制、提供程序行为、快捷方式和故障排除。对于
现在，**Connected** 提供程序和 **Copied to clipboard** 结果确认基本路径正在运行。

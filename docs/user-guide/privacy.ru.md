# Конфиденциальность и данные

GPT-Voice работает с речью, выделенным текстом, учётными данными и настройками браузера. На этой странице описаны
текущие потоки данных и доступные элементы управления для их удаления. Она не заменяет политику
конфиденциальности или условия выбранных вами служб.

## Потоки данных

GPT-Voice отправляет данные за пределы компьютера только при использовании функции, основанной на внешней службе:

| Функция                       | Отправляемые данные                                    | Назначение                                       |
| ----------------------------- | ------------------------------------------------------ | ------------------------------------------------ |
| Расшифровка с **ChatGPT Web** | Подготовленная запись                                  | ChatGPT через сеанс браузера, в который вы вошли |
| Расшифровка с **OpenAI API**  | Подготовленная запись и заданные параметры расшифровки | Конечная точка OpenAI для расшифровки аудио      |
| **Translate**                 | Выделенный текст                                       | Google Translate                                 |
| **Prettify**                  | Выделенный текст и настроенный запрос Prettify         | Настроенная конечная точка Ollama или vLLM       |

Используйте только те учётные записи и конечные точки, которым доверяете, и изучайте их условия обработки данных.
Локальная конечная точка Ollama или vLLM на loopback-интерфейсе оставляет запрос на компьютере, где работает эта
служба; удалённая конечная точка получает текст. Для не-loopback конечной точки Prettify используйте HTTPS.
Браузерные службы могут использовать прокси, настроенный в [настройках Network](settings/network.md).

GPT-Voice записывает успешные результаты расшифровки, перевода и Prettify в системный буфер обмена. Операционная
система и другие приложения, имеющие доступ к буферу, могут сохранить или прочитать это значение; очистите или
замените его после вставки конфиденциального результата.

## Локальные данные и временная память

| Данные                                  | Где и как долго хранятся                                                                                                                                             |
| --------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| История успешных расшифровок            | Локальные данные SQLite в `gpt-voice.sqlite3`: время запроса, идентификатор и имя поставщика, а также текст расшифровки. Записанное аудио не хранится.               |
| Запись, доступная для повторной попытки | Самое последнее подготовленное аудио хранится только в памяти работающего приложения после неудачной расшифровки. Новая запись или перезапуск GPT-Voice удаляет его. |
| Кэш результатов Prettify                | До 20 результатов хранятся в памяти до 60 секунд. Контекст кэша зависит от выделенного текста, запроса и настроек поставщика; при выходе из GPT-Voice он удаляется.  |
| Настройки и идентичность браузера       | Хранятся в локальных данных приложения GPT-Voice и используются при следующих запусках.                                                                              |
| Аутентификация ChatGPT Web              | Локальные данные сеанса и аутентификации браузера. Для удаления используйте элемент управления поставщика.                                                           |

Действия с выделенным текстом временно читают и заменяют содержимое буфера обмена, пока получают выделение. Если
Translate или Prettify завершается ошибкой либо отменяется, GPT-Voice восстанавливает значение буфера, сохранённое до
действия. При успехе в буфере остаётся результат, который можно вставить.

## Учётные данные и ограничения шифрования

Ключи OpenAI API, ключи vLLM API и пароли HTTP/HTTPS-прокси хранятся через Electron safe storage, когда эта защита
доступна. GPT-Voice не возвращает сохранённые значения в окна Settings. Если защищённое хранилище недоступно,
GPT-Voice не может сохранить новый секрет через этот элемент управления.

Это не означает, что зашифрован каждый файл в каталоге данных приложения. В частности, история расшифровок,
обычные настройки и данные сеанса ChatGPT Web имеют собственные правила локального хранения. Не передавайте ключи
API, пароли прокси, сведения о сеансе, продиктованный текст или снимки экрана с этими данными.

## Удаление или сброс данных

Выберите наиболее узкий элемент управления, который подходит в вашей ситуации:

1. В окне History используйте **Clear history**, чтобы навсегда удалить все сохранённые записи расшифровок. См.
   [историю и системный трей](guides/history-and-tray.md).
2. В элементах управления поставщиками расшифровки используйте **Clear session** для ChatGPT Web или
   **Clear API key** для OpenAI API. См. [настройки поставщика](settings/providers.md).
3. Используйте **Clear API key** в [настройках Prettify](settings/prettify.md) и элемент **Clear** для пароля в
   [настройках Network](settings/network.md), когда это применимо.
4. Отдельно замените или очистите системный буфер обмена, если в нём остаётся результат, который вы не хотите
   вставлять.
5. Для полного локального сброса завершите GPT-Voice из трея и удалите сохранённый каталог данных приложения:
   `%APPDATA%\GPT-Voice` в Windows или `~/.config/GPT-Voice` в Linux. Это удалит настройки, историю и сохранённые
   данные поставщиков, управляемые приложением, и потребует повторной настройки после переустановки или запуска.

Удаление локальных данных необратимо. Обычное удаление приложения намеренно сохраняет эти каталоги; описание для
каждой платформы приведено в разделе [установка, обновление или удаление](install.md).

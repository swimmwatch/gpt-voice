# पहला उपयोग: एक प्रदाता को कनेक्ट करें और भाषण को ट्रांसक्राइब करें

[GPT-Voice](install.md) इंस्टॉल करने के बाद, एक ट्रांसक्रिप्शन प्रदाता को कनेक्ट करें, माइक्रोफ़ोन एक्सेस की अनुमति दें, और एक बनाएं
लघु रिकॉर्डिंग. GPT-Voice एक सफल ट्रांसक्रिप्शन को क्लिपबोर्ड पर कॉपी करता है; यह स्वचालित रूप से टाइप नहीं होता है
आप जिस एप्लिकेशन का उपयोग कर रहे थे.

## 1. एक प्रतिलेखन प्रदाता चुनें

GPT-Voice खोलें और कमांड डॉक में **Provider** चयनकर्ता में **ChatGPT Web** या **OpenAI API** चुनें।

| प्रदाता         | आपको क्या चाहिए                                          | पहली बार कार्रवाई                                                 |
| --------------- | -------------------------------------------------------- | ----------------------------------------------------------------- |
| **ChatGPT Web** | एक चैटजीपीटी खाता जिसमें आप साइन इन कर सकते हैं।         | इसे चुनें, फिर **Connect** चुनें और ब्राउज़र साइन-इन पूरा करें।   |
| **OpenAI API**  | आपकी अपनी OpenAI API कुंजी और उपलब्ध API बिलिंग या कोटा। | इसे चुनें, फिर प्रदाता सेटिंग्स खोलने के लिए **Configure** चुनें। |

प्रदाता की उपलब्धता, सीमाएं, बिलिंग और शर्तें आपके द्वारा उपयोग किए जाने वाले खाते द्वारा नियंत्रित की जाती हैं। GPT-Voice बायपास नहीं होता
वे सीमाएँ.

### ChatGPT Web कनेक्ट करें

1. **ChatGPT Web** चुनें।
2. **Connect** चुनें।
3. खुलने वाली ब्राउज़र विंडो में चैटजीपीटी साइन-इन पूरा करें।
4. चैटजीपीटी तैयार होने के बाद, उस लॉगिन विंडो को बंद करें और GPT-Voice पर वापस लौटें।

सत्र तैयार होने पर प्रदाता स्थिति **Connected** में बदल जाती है। GPT-Voice आपके ब्राउज़र सत्र को सहेजता है
स्थानीय ऐप डेटा और बाद में लॉन्च होने पर इसका बैकग्राउंड ब्राउज़र स्वचालित रूप से प्रारंभ हो जाता है। यदि सत्र समाप्त हो जाता है, तो चुनें
**Connect** फिर से साइन इन करने के लिए।

### OpenAI API कॉन्फ़िगर करें

1. **OpenAI API** चुनें।
2. **Configure** चुनें। आप प्रदाता चयनकर्ता के बगल में प्रदाता-सेटिंग्स नियंत्रण का भी उपयोग कर सकते हैं।
3. अपनी OpenAI API कुंजी चिपकाएँ।
4. वैकल्पिक रूप से एक प्रतिलेखन भाषा, संकेत या तापमान चुनें।
5. **Save** चुनें।

प्रतिलेखन मॉडल `whisper-1` पर निर्धारित है। सफल सेव के बाद, GPT-Voice प्रदाता को इस रूप में दिखाता है
**Connected** और रिपोर्ट करता है कि प्रदाता कॉन्फ़िगर किया गया है। एप्लिकेशन इलेक्ट्रॉन का उपयोग करके एपीआई कुंजी को स्थानीय रूप से संग्रहीत करता है
उपलब्ध होने पर सुरक्षित भंडारण; कुंजी को इंटरफ़ेस में वापस नहीं दिखाया गया है। OpenAI API ट्रांसक्रिप्शन का उपयोग नहीं करता है
ब्राउज़र.

## 2. माइक्रोफ़ोन एक्सेस की अनुमति दें

पहली रिकॉर्डिंग आपके ऑपरेटिंग सिस्टम से माइक्रोफ़ोन की अनुमति मांगती है। GPT-Voice के लिए पहुंच की अनुमति दें, फिर वापस लौटें
कमांड डॉक. यदि पहुंच अस्वीकृत है या कोई माइक्रोफ़ोन उपलब्ध नहीं है, तो स्थिति दिखाती है **त्रुटि: पहुंच नहीं सका
माइक्रोफ़ोन** और कोई ऑडियो नहीं भेजा गया है। प्रयास करने से पहले अपने ऑपरेटिंग सिस्टम के गोपनीयता नियंत्रण में अनुमति सक्षम करें
पुनः.

## 3. पहली रिकॉर्डिंग करें

1. एक प्रदाता चुनें जो **Connected** दिखाता है।
2. **Start recording** चुनें या प्रदर्शित रिकॉर्डिंग शॉर्टकट दबाएं (डिफ़ॉल्ट `F9` है)।
3. एक छोटा वाक्य बोलें. स्थिति **Recording** में बदल जाती है।
4. **Stop recording** (डिफ़ॉल्ट `F10`) चुनें। GPT-Voice भेजते समय स्थिति को **Transcribing** में बदल देता है
   चयनित प्रदाता को ऑडियो कैप्चर किया गया।
5. **Copied to clipboard** की प्रतीक्षा करें, फिर परिणाम की पुष्टि करने के लिए किसी भी टेक्स्ट फ़ील्ड में पेस्ट करें।

आप कमांड डॉक से किसी सक्रिय रिकॉर्डिंग को रोक भी सकते हैं, फिर से शुरू कर सकते हैं या रद्द भी कर सकते हैं। रद्द करने से सक्रिय हटा दिया जाता है
प्रतिलेखन के लिए भेजने के बजाय रिकॉर्डिंग करना।

## यदि पहला प्रतिलेखन काम नहीं करता है

- यदि **Connected** नहीं दिखाया गया है, तो प्रदाता नियंत्रण को फिर से खोलें और साइन-इन पूरा करें या एक वैध एपीआई कुंजी सहेजें। ChatGPT Web के लिए, स्थिति इसके बजाय कह सकती है कि ब्राउज़र आरंभीकरण विफल रहा या सत्र समाप्त हो गया।
- यदि स्थिति माइक्रोफ़ोन त्रुटि की रिपोर्ट करती है, तो GPT-Voice को ऑपरेटिंग सिस्टम में अपने माइक्रोफ़ोन का उपयोग करने की अनुमति दें और एक बनाएं
  नई रिकॉर्डिंग.
- यदि स्थिति **Transcription failed** या **Transcription error** रिपोर्ट करती है, तो चयनित प्रदाता के खाते की जांच करें,
  कनेक्शन, और सीमाएँ, फिर पुनः प्रयास करें। विफल प्रतिलेखन को क्लिपबोर्ड पर कॉपी नहीं किया जाता है।

अगले गाइड पेज में रिकॉर्डिंग नियंत्रण, प्रदाता व्यवहार, शॉर्टकट और समस्या निवारण के बारे में अधिक विस्तार से बताया गया है। के लिए
अब, एक **Connected** प्रदाता और एक **Copied to clipboard** परिणाम पुष्टि करते हैं कि मूल पथ काम कर रहा है।

# Перевод и Prettify для выделенного текста

GPT-Voice может работать с текстом, выделенным в другом приложении. **Translate** отправляет выделение в Google
Translate на выбранный вами целевой язык. **Prettify** отправляет его настроенному поставщику обработки текста, чтобы
улучшить текст согласно его подсказке. Ни одно из этих действий не вставляет текст в другое приложение: при успехе
скопируйте результат из системного буфера обмена и вставьте его туда, где он нужен.

## Включите действие и выберите его сочетание клавиш

Откройте **Settings** и выберите **Shortcuts**. В строках **Translate** и **Prettify** есть переключатель включения и
элемент **Change** для глобального сочетания клавиш. Оба действия включены по умолчанию; сочетание по умолчанию для
Translate — `F11`, для Prettify — `F12`. После изменения сохраните настройки.

Выберите сочетание, которое не конфликтует с другим сочетанием GPT-Voice или используемым вами программным
обеспечением. Отключённое действие не выполняется при нажатии сочетания. Действие также ждёт завершения активной
записи, и GPT-Voice выполняет одновременно только одно действие с выделенным текстом.

## Переведите выделенный текст

1. В приложении с текстом выделите текст, который хотите перевести.
2. На панели команд GPT-Voice выберите **Target language**: английский, русский, украинский или белорусский.
3. Нажмите включённое сочетание Translate (по умолчанию `F11`).
4. Дождитесь уведомления об успехе, затем вставьте переведённый текст из буфера обмена.

GPT-Voice копирует выделенный текст с помощью обычного действия копирования операционной системы, затем отправляет
его в Google Translate. Перевод — внешняя служба: выделенный текст отправляется в Google Translate, поэтому не
используйте это действие для текста, которым вам не разрешено делиться с этой службой.

Если текста не выделено, копирование нельзя автоматизировать или служба не может вернуть результат, GPT-Voice
показывает безопасное сообщение об ошибке и восстанавливает значение буфера обмена, которое было до действия. В
Linux приложение также может использовать буфер выделения, когда обычное копирование не удаётся. При успешном
переводе системный буфер обмена заменяется переведённым результатом.

## Улучшите выделенный текст с помощью Prettify

Перед использованием Prettify откройте **Settings** и выберите **Prettify**. Выберите **Ollama** или **vLLM**,
укажите адрес и модель поставщика и сохраните действительную конфигурацию. Для vLLM также может требоваться API-ключ.

1. Выделите в редактируемом приложении не более 16,000 символов.
2. Нажмите включённое сочетание Prettify (по умолчанию `F12`).
3. Дождитесь уведомления **Text prettified**, затем вставьте результат из буфера обмена.

Ollama и vLLM — зависимости, которыми управляет пользователь: GPT-Voice не устанавливает, не размещает и не
управляет ни одной из этих служб. Локальный конечный адрес обратной петли оставляет запрос на машине, где работает
служба. Удалённый конечный адрес получает выделенный текст; используйте поставщика, которому доверяете, соблюдайте
его правила обработки данных и применяйте HTTPS для нелокального конечного адреса.

Если текст не выделен, выделение превышает 16,000 символов, настроенный поставщик недоступен или не возвращает
пригодный результат, GPT-Voice сообщает о сбое и восстанавливает предыдущее содержимое буфера обмена. Пока выполняется
запрос Prettify, настроенное сочетание отмены (по умолчанию `Escape`) отменяет его и восстанавливает это значение
буфера обмена.

## Буфер обмена и одновременные действия

Translate и Prettify намеренно взаимоисключающие. Если запустить одно действие, пока другое уже выполняется, новый
запрос пропускается; дождитесь статуса или уведомления перед следующей попыткой. При успехе результат заменяет буфер
обмена; при неудачном или отменённом запросе GPT-Voice восстанавливает значение буфера, захваченное до чтения
выделения. Всегда проверяйте вставленный результат, особенно если исходный текст содержит имена, код или другие
точные значения.

# 翻译并美化选定的文本

GPT-Voice 可以作用于在另一个应用程序中选择的文本。 **Translate** 将选择发送到 Google Translate
您选择的目标语言。 **Prettify** 将其发送到您配置的文本处理提供商以改进文本
根据其提示。两个操作都不会粘贴到另一个应用程序中：成功后，从系统复制结果
剪贴板并将其粘贴到您需要的地方。

## 启用一个操作并选择其快捷方式

打开 **Settings** 并选择 **Shortcuts**。 **Translate** 和 **Prettify** 行各有一个使能开关和一个
**Change** 控制其全局快捷方式。默认情况下启用这两个操作；他们的默认快捷键是 `F11`
用于翻译，`F12` 用于美化。更改后保存设置。

选择不与其他 GPT-Voice 快捷方式或您使用的软件冲突的快捷方式。禁用的动作
按下其快捷键时不运行。操作也会等待，直到活动记录结束，并且 GPT-Voice 运行
一次只能执行一个选定的文本操作。

## 翻译所选内容

1. 在包含文本的应用程序中，选择要翻译的文本。
2. 在GPT-VoiceCommand Dock中，选择**Target language**：英语、俄语、乌克兰语或白俄罗斯语。
3. 按已启用的翻译快捷键（默认情况下为 `F11`）。
4. 等待成功通知，然后从剪贴板粘贴翻译后的文本。

GPT-Voice 使用操作系统的正常复制操作复制所选文本，然后将其提交给 Google
翻译。翻译是一项外部服务：所选文本将发送到 Google Translate，因此请勿使用此操作
对于文本，您不得与该服务共享。

如果没有选定的文本，则无法自动进行复制，或者服务无法返回结果，GPT-Voice 会报告
安全错误消息并恢复操作之前存在的剪贴板值。在 Linux 上，还可以使用
当正常复制操作失败时选择剪贴板。成功的翻译将系统剪贴板替换为
翻译结果。

## 美化选区

使用 Prettify 之前，请打开 **Settings** 并选择 **Prettify**。选择 **Ollama** 或 **vLLM**，设置提供商
地址和型号，并保存有效的配置。 vLLM 还可能需要 API 密钥。

1. 在您正在编辑的应用程序中选择最多 16,000 个字符。
2. 按启用的美化快捷键（默认情况下为`F12`）。
3. 等待 **Text prettified** 通知，然后粘贴剪贴板中的结果。

Ollama 和 vLLM 是用户操作的依赖项：GPT-Voice 不安装、托管或管理任一服务。本地人
环回端点将请求保留在运行该服务的计算机上。远程端点接收所选文本；
使用您信任的提供商，遵循其数据处理策略，并对非本地端点使用 HTTPS。如果未选择任何文本，则选择超过 16,000 个字符，则无法访问配置的提供程序，或者
没有返回可用的结果，GPT-Voice报告失败并恢复以前的剪贴板内容。在美化的同时
请求正在运行，配置的取消快捷方式（默认 `Escape`）会取消它并恢复该剪贴板值。

## 剪贴板和并发操作

Translation 和 Prettify 是故意相互排斥的。当另一个已经开始工作时启动一个会跳过
新请求，因此请等待状态或通知，然后重试。成功后，结果将替换
剪贴板；如果请求失败或取消，GPT-Voice 会恢复在读取之前捕获的剪贴板值
选择。请务必检查粘贴的结果，尤其是当源文本包含名称、代码或其他精确值时。

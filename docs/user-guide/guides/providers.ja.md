# 文字起こしプロバイダーの選択と管理

GPT-Voice は、一度に 1 つのアクティブな文字起こしプロバイダーを使用します。 Command Dock の **Provider** セレクターから選択します。セレクターを切り替えると、次の録音を受信するプロバイダーが変わります。プロバイダー間でセッション、API キー、請求、アカウント アクセスを転送することはありません。

| プロバイダー    | 認証                                           | 音声の送信先                                |
| --------------- | ---------------------------------------------- | ------------------------------------------- |
| **ChatGPT Web** | サインインした ChatGPT ブラウザー セッション。 | サインインした ChatGPT セッションを通じて。 |
| **OpenAI API**  | あなたが提供する OpenAI API キー。             | OpenAI の音声文字起こしエンドポイント。     |

使用を許可されているアカウントのみを使用してください。可用性、請求、クォータ、使用制限、およびサービス条件は、プロバイダーのアカウントとサービスによって設定されます。 GPT-Voice はそれらをバイパスしません。

## ChatGPT Web

**ChatGPT Web** を選択し、次に **Connect** を選択します。 GPT-Voice は ChatGPT でブラウザのログイン ウィンドウを開きます。そこでサインインを完了し、ログイン ページを閉じます。 GPT-Voice は、結果として得られるブラウザ セッションをローカルに保存し、セッションがまだ使用可能な場合に、その後の起動時にバックグラウンド ブラウザを起動します。

セッションの準備が完了すると、プロバイダーには **Connected** と表示されます。セッションの有効期限が切れた場合、GPT-Voice によって使用できなくなった保存されたセッションが削除されるため、**Connect** を再度選択する必要があります。転写リクエストは、有効期間の短いアクセス トークンを 1 回更新することもできます。これは、基礎となるセッションが有効でなくなった場合に、新しいサインインを置き換えるものではありません。

GPT-Voice からサインアウトするには、プロバイダー設定を開いて **Clear authentication** を使用します。これにより、GPT-Voice の保存された ChatGPT ブラウザ セッション データとキャッシュされたアクセス トークンが削除されます。アカウント、サブスクリプション、または他のブラウザーやデバイスによって保持されているセッションは管理されません。

## OpenAI API

**OpenAI API** を選択し、**Configure** を選択するか、セレクターの横にあるプロバイダー設定コントロールを開きます。独自の API キーを貼り付けてフォームを保存します。 GPT-Voice は固定転写モデル `whisper-1` を使用します。このプロバイダーが別の転写モデルを選択することはできません。

プロバイダー設定では、次のことも選択できます。

- **Language**: 自動検出 (デフォルト)、英語、ロシア語、ウクライナ語、またはベラルーシ語。
- **Prompt**: 文字起こしリクエストとともに送信されるオプションのガイダンス。
- **Temperature**: `0` から `1` までの値。デフォルトは `0` です。

GPT-Voice は、保護が利用可能な場合、Electron safe storage を使用して API キーをローカルに保存し、保存されたキーをインターフェイスに表示することはありません。安全なストレージが利用できない場合、その保護なしで新しいキーを保存するのではなく、新しいキーの保存は失敗します。 OpenAI API プロバイダーはブラウザのログイン ウィンドウを使用しません。保存された API キーを削除するには、プロバイダー設定で **Clear authentication** を使用します。非シークレットのトランスクリプション オプションは、次に構成するキーに対して引き続き使用できますが、有効なキーが再度保存されるまでプロバイダーは接続されません。

## プロバイダーの切り替えまたは回復

録画を開始する前にプロバイダーを切り替えることができます。記録する前に、新しく選択したプロバイダーが **Connected** であることを確認してください。構成された ChatGPT Web セッションと構成された OpenAI API キーは独立しています。

転写が失敗した場合:

1. 選択したプロバイダーがまだ接続されており、そのアカウントが文字起こしを使用できることを確認します。
2. **ChatGPT Web** の場合、セッションの有効期限が切れた場合、またはブラウザが初期化できなかった場合は、再接続します。
3. **OpenAI API** については、API キー、請求または割り当て、プロバイダーのサービス ステータスを確認し、必要な修正を保存します。
4. [記録と転写](transcription.md)に戻って、最後に準備されたキャプチャを再試行するか、新しい記録を開始します。

プロバイダーの設定は、今後のリクエストに影響します。キャンセルされた録画を回復したり、プロバイダー側​​のアクセス、クォータ、ポリシー、サービス エラーを上書きしたりすることはできません。

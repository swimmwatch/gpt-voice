# 録音と文字起こし

録音する前に、**Connected** と表示されるプロバイダーに接続し、GPT-Voice によるマイクの使用を許可してください。参照
[最初に使用する](../getting-started.md) セットアップが完了していない場合は、[プロバイダー ガイド](providers.md)を確認してください
接続とアカウントの詳細については、

## 記録ライフサイクル

Command Dock から、または設定された録画ショートカット (デフォルトは `F9`) から録画を開始します。ステータス
マイクキャプチャが開始されると、 **Ready** から **Recording** に変わります。録音中の主なアクションは次のとおりです。
**Stop recording** (デフォルトは `F10`)。

| アクション          | 利用可能な場合                                     | 何が起こるか                                                                                                                                             |
| ------------------- | -------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Start recording** | GPT-Voice はアイドル状態です。                     | マイクへのアクセスを要求し、新しいキャプチャを開始します。新しいキャプチャを開始すると、以前のキャプチャから再試行可能なオーディオがすべて消去されます。 |
| **Pause**           | キャプチャは記録中です。                           | 現在のキャプチャを送信せずに一時停止します。                                                                                                             |
| **Resume**          | キャプチャは一時停止されます。                     | 同じキャプチャを継続します。                                                                                                                             |
| **Stop recording**  | キャプチャは記録中または一時停止中です。           | キャプチャを終了し、音声を準備して、選択したプロバイダーに送信します。                                                                                   |
| **Cancel**          | GPT-Voice は開始中、録画中、または一時停止中です。 | アクティブなキャプチャを停止して破棄します。転写のために送信されることはありません。                                                                     |

GPT-Voice が停止中、音声の準備中、文字起こし中、または再試行中は、現在の操作が完了するまで待ちます
別の録音を開始する前に。 Command Dock はこの間の処理状態を示します。

## 停止するとどうなるか

**Stop recording** の後、GPT-Voice はキャプチャされた音声を準備し、**Transcribing** を表示します。準備されたものを送信します
Command Dock で選択したプロバイダーへの音声:

- **ChatGPT Web** は、サインインしている ChatGPT ブラウザ セッションを通じて音声を送信します。
- **OpenAI API** は、構成した API キーを使用して OpenAI の文字起こしエンドポイントに音声を送信します。

プロバイダーの可用性、アカウント アクセス、請求、割り当て、およびサービス条件は、そのプロバイダー アカウントによって制御されます。
GPT-Voice はこれらのコントロールをバイパスしません。

成功すると、GPT-Voice は返されたテキストをシステムのクリップボードにコピーし、ステータスを **コピー済み に変更します。
クリップボード**にアクセスし、成功通知を要求します。使用していたアプリケーションにテキストを貼り付けます。 GPT-Voiceもテキスト、プロバイダー名、リクエスト時刻をローカルの文字起こし履歴に保存します。履歴コントロールは
別に文書化されています。

## 失敗した文字起こしを再試行します

GPT-Voice が空ではないキャプチャを準備した後、その準備されたオーディオを最新の再試行可能なオーディオとしてメモリに保持します。
録音中。転写リクエストが失敗した場合は、GPT-Voice がアイドル状態のときに、設定された再試行転写アクションを使用して、
同じ準備された音声を再度送信します。再試行してもマイクは再度録音されません。

この再試行コピーは意図的に一時的なものです。

- 新しい録音を開始する前にクリアされます。
- 記録中、処理中、またはリトライ中は使用できません。
- 実行中のアプリケーションのメモリ内にのみ保持されます。 GPT-Voice を再起動すると削除されます。

再試行は、接続、セッション、またはプロバイダーの問題を修正した後で、失敗した送信を繰り返す方法です。そうではありません
プロバイダーの制限を変更したり、期限切れのアカウント セッションを復元したり、プロバイダーがリクエストを受け入れることを保証したりできます。

## 録音または文字起こしが失敗した場合

- **Could not access microphone** は、GPT-Voice がオーディオ ストリームを取得できなかったことを意味します。オペレーティング システムを確認してください
  マイクのプライバシーを許可し、マイクが接続されていることを確認してから、新しい録音を開始します。
- **Transcription failed** は、プロバイダーが失敗した結果を返したことを意味します。プロバイダーアカウント、ネットワーク、
  再試行する前に、サービスの可用性と適用可能な制限を確認してください。
- **Transcription error** は、GPT-Voice が準備またはリクエストを完了できなかったことを意味します。障害通知と
  Command Dock ステータスは、ユーザーに向けた安全なエラー メッセージを提供します。失敗したテキストはクリップボードにコピーされません。
- ChatGPT Web セッションの有効期限が切れた場合は、再試行する前に再接続してください。 OpenAI API キーが見つからないか拒否された場合、
  再試行する前にプロバイダー設定で修正してください。

アクティブな記録のキャンセルは、失敗した送信とは異なります。キャンセルすると、キャプチャが完了する前に破棄されます。
準備または送信されたため、その録音に対して再試行可能な文字起こしはありません。

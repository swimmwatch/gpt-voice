# Выбар пастаўшчыка расшыфроўкі і кіраванне ім

GPT-Voice выкарыстоўвае толькі аднаго актыўнага пастаўшчыка расшыфроўкі за раз. Абярыце яго ў селектары
**Provider** у Command Dock. Змена селектара вызначае, які пастаўшчык атрымае наступны запіс; сеансы, ключы API,
аплата і доступ да ўліковага запісу паміж пастаўшчыкамі не пераносяцца.

| Пастаўшчык      | Аўтэнтыфікацыя                              | Куды адпраўляецца аўдыя                      |
| --------------- | ------------------------------------------- | -------------------------------------------- |
| **ChatGPT Web** | Сеанс ChatGPT у браўзеры, у які вы ўвайшлі. | Праз сеанс ChatGPT, у які вы ўвайшлі.        |
| **OpenAI API**  | Прадастаўлены вамі ключ OpenAI API.         | На канечную кропку расшыфроўкі аўдыя OpenAI. |

Выкарыстоўвайце толькі ўліковы запіс, якім вам дазволена карыстацца. Даступнасць, аплата, квоты, абмежаванні
выкарыстання і ўмовы службы вызначаюцца ўліковым запісам і службай пастаўшчыка. GPT-Voice не абыходзіць іх.

## ChatGPT Web

Абярыце **ChatGPT Web**, а затым **Connect**. GPT-Voice адкрые ў браўзеры акно ўваходу ў ChatGPT. Завяршыце ўваход
там, потым зачыніце старонку ўваходу. GPT-Voice захоўвае атрыманы сеанс браўзера лакальна і запускае фонавы браўзер
пры наступных запусках, калі сеанс яшчэ прыдатны.

Калі сеанс гатовы, пастаўшчык паказвае **Connected**. Калі сеанс скончыўся, GPT-Voice выдаляе непрыдатны захаваны
сеанс, і вам трэба зноў абраць **Connect**. Запыт расшыфроўкі можа таксама адзін раз абнавіць свой кароткачасовы
токен доступу; гэта не замяняе новы ўваход, калі базавы сеанс больш не дзейнічае.

Каб выйсці з GPT-Voice, адкрыйце налады пастаўшчыка і выкарыстайце **Clear authentication**. Гэта выдаляе захаваныя
GPT-Voice даныя сеанса ChatGPT у браўзеры і кэшаваны токен доступу. Гэта дзеянне не кіруе вашым уліковым запісам,
падпіскай або сеансамі ў іншых браўзерах і на іншых прыладах.

## OpenAI API

Абярыце **OpenAI API** і **Configure** або адкрыйце элемент налад пастаўшчыка побач з селектарам. Устаўце свой ключ
API і захавайце форму. GPT-Voice выкарыстоўвае фіксаваную мадэль расшыфроўкі `whisper-1`; для гэтага пастаўшчыка
нельга абраць іншую мадэль расшыфроўкі.

У наладах пастаўшчыка таксама можна выбраць:

- **Language**: аўтаматычнае вызначэнне (па змаўчанні), англійскую, рускую, украінскую або беларускую мову.
- **Prompt**: неабавязковую падказку, што адпраўляецца з запытам расшыфроўкі.
- **Temperature**: значэнне ад `0` да `1`; па змаўчанні — `0`.

GPT-Voice захоўвае ключ API лакальна праз Electron safe storage, калі такая абарона даступная, і ніколі не паказвае
захаваны ключ у інтэрфейсе. Калі бяспечнае сховішча недаступнае, захаванне новага ключа не ўдаецца, а ключ не
захоўваецца без гэтай абароны. Пастаўшчык OpenAI API не выкарыстоўвае акно ўваходу ў браўзеры.

Выкарыстоўвайце **Clear authentication** у наладах пастаўшчыка, каб выдаліць захаваны ключ API. Несакрэтныя
параметры расшыфроўкі застануцца для наступнага наладжанага ключа, але пастаўшчык не падлучаны, пакуль сапраўдны
ключ не будзе захаваны зноў.

## Змяніце пастаўшчыка або аднавіце працу

Вы можаце змяніць пастаўшчыка да пачатку запісу. Перад запісам пераканайцеся, што новаабраны пастаўшчык мае стан
**Connected**: наладжаны сеанс ChatGPT Web і наладжаны ключ OpenAI API не залежаць адзін ад аднаго.

Калі расшыфроўка не ўдалася:

1. Праверце, што выбраны пастаўшчык усё яшчэ падлучаны і яго ўліковы запіс можа выкарыстоўваць расшыфроўку.
2. Для **ChatGPT Web** падлучыцеся зноў, калі сеанс скончыўся або не ўдалося ініцыялізаваць браўзер.
3. Для **OpenAI API** праверце ключ API, аплату або квоту і стан службы пастаўшчыка, затым захавайце патрэбнае
   выпраўленне.
4. Вярніцеся да [запісу і расшыфроўкі](transcription.md), каб паўтарыць апошні падрыхтаваны захоп, або пачніце новы
   запіс.

Налады пастаўшчыка ўплываюць на будучыя запыты. Яны не могуць аднавіць адменены запіс або абысці памылкі доступу,
квоты, палітыкі ці службы на баку пастаўшчыка.

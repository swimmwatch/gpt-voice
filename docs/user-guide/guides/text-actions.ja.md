# 選択したテキストを翻訳して整形する

GPT-Voice は、別のアプリケーションで選択されたテキストに作用できます。 **Translate** は選択内容を Google Translate に送信します
あなたが選択したターゲット言語。 **Prettify** は、テキストを改善するために、設定されたテキスト処理プロバイダーにそれを送信します。
そのプロンプトに従って。どちらのアクションも他のアプリケーションには貼り付けられません。成功すると、システムから結果がコピーされます。
クリップボードを選択し、必要な場所に貼り付けます。

## アクションを有効にし、そのショートカットを選択します

**Settings** を開き、**Shortcuts** を選択します。 **Translate** 行と **Prettify** 行にはそれぞれイネーブル スイッチと
グローバル ショートカットの **Change** コントロール。どちらのアクションもデフォルトで有効になっています。デフォルトのショートカットは `F11` です
翻訳の場合は `F12`] の場合は Prettify です。設定を変更したら保存します。

別の GPT-Voice ショートカットや使用するソフトウェアと競合しないショートカットを選択してください。無効化されたアクション
ショートカットを押しても実行されません。アクションは、アクティブな記録が終了し、GPT-Voice が実行されるまで待機します。
選択テキスト アクションは一度に 1 つだけです。

## 選択範囲を翻訳する

1. テキストを含むアプリケーションで、翻訳するテキストを選択します。
2. GPT-Voice Command Dock で、**Target language**: 英語、ロシア語、ウクライナ語、またはベラルーシ語を選択します。
3. 有効になっている翻訳ショートカット (デフォルトでは `F11`) を押します。
4. 成功通知を待ってから、翻訳されたテキストをクリップボードから貼り付けます。

GPT-Voice は、オペレーティング システムの通常のコピー アクションを使用して選択されたテキストをコピーし、Google に送信します。
翻訳します。翻訳は外部サービスです。選択したテキストは Google Translate に送信されるため、このアクションは使用しないでください
そのサービスと共有することが許可されていないテキストの場合。

選択されたテキストがない場合、コピーを自動化できない場合、またはサービスが結果を返せない場合、GPT-Voice は
安全なエラー メッセージが表示され、アクションの前に存在していたクリップボードの値が復元されます。 Linux では、
通常のコピー操作が失敗した場合の選択クリップボード。翻訳が成功すると、システム クリップボードが
翻訳した結果。

## 選択範囲を美しくする

Prettify を使用する前に、**Settings** を開いて **Prettify** を選択してください。 **Ollama** または **vLLM** のいずれかを選択し、プロバイダーを設定します
アドレスとモデルを入力し、有効な設定を保存します。 vLLM には API キーが必要な場合もあります。

1. 編集中のアプリケーションで最大 16,000 文字を選択します。
2. 有効になっている Prettify ショートカット (デフォルトでは `F12`) を押します。
3. **Text prettified** 通知を待ってから、クリップボードから結果を貼り付けます。

Ollama と vLLM はユーザー操作の依存関係です。 GPT-Voice はどちらのサービスもインストール、ホスト、管理しません。地元の人
ループバック エンドポイントは、そのサービスを実行しているマシン上でリクエストを保持します。リモート エンドポイントは選択されたテキストを受信します。
信頼できるプロバイダーを使用し、そのデータ処理ポリシーに従い、非ローカル エンドポイントには HTTPS を使用します。テキストが選択されていない場合、選択内容が 16,000 文字を超えている場合、構成されたプロバイダーに到達できない場合、または
使用可能な結果が返されない場合、GPT-Voice は失敗を報告し、以前のクリップボードの内容を復元します。きれいにしながら
リクエストが実行されている場合、設定されたキャンセル ショートカット (デフォルト `Escape`) によってリクエストがキャンセルされ、そのクリップボード値が復元されます。

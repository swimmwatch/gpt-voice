# Запіс і расшыфроўка

Перад запісам падлучыце пастаўшчыка са станам **Connected** і дазвольце GPT-Voice выкарыстоўваць мікрафон. Калі вы
яшчэ не прайшлі гэты крок, глядзіце [першы запуск](../getting-started.md), а звесткі пра ўваход і ўліковы запіс — у
[кіраўніцтве па пастаўшчыках](providers.md).

## Жыццёвы цыкл запісу

Пачніце запіс з Command Dock або наладжаным спалучэннем клавіш (па змаўчанні `F9`). Калі захоп з мікрафона пачнецца,
стан зменіцца з **Ready** на **Recording**. Падчас запісу галоўнае дзеянне — **Stop recording** (па змаўчанні `F10`).

| Дзеянне             | Калі даступнае                         | Што адбываецца                                                                                                          |
| ------------------- | -------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| **Start recording** | GPT-Voice бяздзейнічае.                | Запытвае доступ да мікрафона і пачынае новы захоп. Новы захоп ачышчае аўдыя, даступнае для папярэдняй паўторнай спробы. |
| **Pause**           | Ідзе запіс.                            | Прыпыняе бягучы захоп без адпраўкі.                                                                                     |
| **Resume**          | Захоп прыпынены.                       | Працягвае той самы захоп.                                                                                               |
| **Stop recording**  | Ідзе або прыпынены запіс.              | Завяршае захоп, рыхтуе аўдыя і адпраўляе яго выбранаму пастаўшчыку.                                                     |
| **Cancel**          | Запіс запускаецца, ідзе або прыпынены. | Спыняе і адкідвае актыўны захоп; ён не адпраўляецца на расшыфроўку.                                                     |

Пакуль GPT-Voice спыняе запіс, рыхтуе аўдыя, расшыфроўвае або паўтарае запыт, пачакайце завяршэння бягучай аперацыі,
перш чым пачынаць новы запіс. У гэты час Command Dock паказвае стан апрацоўкі.

## Што адбываецца пасля спынення

Пасля **Stop recording** GPT-Voice рыхтуе захопленае аўдыя і паказвае **Transcribing**. Ён адпраўляе падрыхтаванае
аўдыя пастаўшчыку, абранаму ў Command Dock:

- **ChatGPT Web** адпраўляе аўдыя праз сеанс ChatGPT у браўзеры, у які вы ўвайшлі.
- **OpenAI API** адпраўляе аўдыя на канечную кропку расшыфроўкі OpenAI з наладжаным вамі ключом API.

Даступнасць пастаўшчыка, доступ да ўліковага запісу, аплата, квоты і ўмовы службы вызначаюцца адпаведным
пастаўшчыком. GPT-Voice не абыходзіць гэтыя абмежаванні.

Пры поспеху GPT-Voice капіруе вернуты тэкст у сістэмны буфер абмену, змяняе стан на **Copied to clipboard** і
запытвае апавяшчэнне аб поспеху. Устаўце тэкст у праграму, якой карысталіся. GPT-Voice таксама захоўвае тэкст, імя
пастаўшчыка і час запыту ў лакальнай гісторыі расшыфровак; кіраванне гісторыяй апісана асобна.

## Паўторыце няўдалую расшыфроўку

Пасля таго як GPT-Voice падрыхтаваў непусты захоп, ён трымае гэта падрыхтаванае аўдыя ў памяці як апошні запіс,
даступны для паўторнай спробы. Калі запыт на расшыфроўку не ўдаўся, выкарыстоўвайце наладжанае дзеянне паўторнай
расшыфроўкі, калі GPT-Voice бяздзейнічае, каб зноў адправіць тое самае падрыхтаванае аўдыя.

Паўтор не запісвае мікрафон наноў.

Гэтая копія для паўтору наўмысна часовая:

- Яна ачышчаецца перад пачаткам новага запісу.
- Яна недаступная падчас запісу, апрацоўкі або паўтору.
- Яна захоўваецца толькі ў памяці запушчанай праграмы; перазапуск GPT-Voice яе выдаляе.

Паўтор дазваляе паўтарыць адну няўдалую адпраўку пасля выпраўлення праблемы з падключэннем, сеансам або
пастаўшчыком. Ён не змяняе абмежаванні пастаўшчыка, не аднаўляе пратэрмінаваны сеанс і не гарантуе, што пастаўшчык
прыме запыт.

## Калі запіс або расшыфроўка не ўдаюцца

- **Could not access microphone** азначае, што GPT-Voice не змог атрымаць аўдыяпаток. Праверце дазвол аперацыйнай
  сістэмы на мікрафон, пераканайцеся, што мікрафон падлучаны, і пачніце новы запіс.
- **Transcription failed** азначае, што пастаўшчык вярнуў няўдалы вынік. Праверце ўліковы запіс пастаўшчыка, сетку,
  даступнасць службы і адпаведныя абмежаванні перад паўторам.
- **Transcription error** азначае, што GPT-Voice не здолеў завяршыць падрыхтоўку або запыт. Апавяшчэнне пра памылку і
  стан Command Dock даюць бяспечнае карыстальніцкае паведамленне; няўдалы тэкст не капіруецца ў буфер абмену.
- Калі сеанс ChatGPT Web скончыўся, падлучыце яго зноў перад паўторам. Калі ключ OpenAI API адсутнічае або адхілены,
  выпраўце яго ў наладах пастаўшчыка перад паўторам.

Адмена актыўнага запісу адрозніваецца ад няўдалай адпраўкі: адмена адкідвае захоп да яго падрыхтоўкі і адпраўкі,
таму для такога запісу няма расшыфроўкі, даступнай для паўтору.

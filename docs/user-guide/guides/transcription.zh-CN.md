# 记录并转录

录制之前，请连接显示 **Connected** 的提供商并允许 GPT-Voice 使用您的麦克风。参见
[如果您尚未完成该设置，请首先使用](../getting-started.md)，或查看[提供商指南](providers.md)
有关连接和帐户详细信息。

## 记录生命周期

从 Command Dock 或使用配置的录制快捷方式（默认为 `F9`）开始录制。现状
麦克风捕获开始后，从 **Ready** 更改为 **Recording**。录音时，主要动作是
**Stop recording**（默认`F10`）。

| 行动                | 什么时候可用                     | 会发生什么                                                                     |
| ------------------- | -------------------------------- | ------------------------------------------------------------------------------ |
| **Start recording** | GPT-Voice 空闲。                 | 请求麦克风访问并开始新的捕获。开始新的捕获会清除前一个捕获中的所有可重试音频。 |
| **Pause**           | 捕获正在记录。                   | 暂停当前​​捕获而不提交。                                                       |
| **Resume**          | 捕获已暂停。                     | 继续相同的捕获。                                                               |
| **Stop recording**  | 捕捉正在录制或暂停。             | 结束采集，准备音频，并将其发送给选定的提供商。                                 |
| **Cancel**          | GPT-Voice 正在开始、录制或暂停。 | 停止并放弃主动捕获；它不会被发送进行转录。                                     |

当 GPT-Voice 正在停止、准备音频、转录或重试时，等待当前操作完成
在开始另一次录音之前。 Command Dock表示此时的处理状态。

## 停止后会发生什么

**Stop recording**之后，GPT-Voice准备捕获的音频并显示**Transcribing**。它发送准备好的
音频发送至 Command Dock 中选择的提供商：

- **ChatGPT Web** 通过登录的 ChatGPT 浏览器会话发送音频。
- **OpenAI API** 使用您配置的 API 密钥将音频发送到 OpenAI 的转录端点。

提供商可用性、帐户访问、计费、配额和服务条款均由该提供商帐户控制。
GPT-Voice 不会绕过这些控件。

成功后，GPT-Voice 将返回的文本复制到系统剪贴板，将状态更改为 **已复制到
剪贴板**，并请求成功通知。将文本粘贴到您正在使用的应用程序中。 GPT-Voice也将文本、提供者名称和请求时间保存在其本地转录历史记录中；历史控件是
单独记录。

## 重试失败的转录

GPT-Voice 准备好非空捕获后，会将准备好的音频保留在内存中作为最近的可重试音频
录音。如果转录请求失败，请在 GPT-Voice 空闲时使用配置的重试转录操作
再次发送相同的准备好的音频。重试不会再次录制麦克风。

此重试副本故意是临时的：

- 在开始新录音之前它会被清除。
- 录制、处理或重试时不可用。
- 它仅保存在正在运行的应用程序的内存中；重新启动 GPT-Voice 将其删除。

重试是在更正连接、会话或提供程序问题后重复失败的提交的一种方法。它不
更改提供商限制、恢复过期的帐户会话或保证提供商将接受请求。

## 如果录音或转录失败

- **Could not access microphone** 表示 GPT-Voice 无法获取音频流。检查操作系统的
  麦克风隐私权限，确认麦克风已连接，然后开始新的录音。
- **Transcription failed** 表示提供商返回不成功的结果。检查提供商帐户、网络、
  服务可用性以及重试之前的适用限制。
- **Transcription error** 表示 GPT-Voice 无法完成准备或请求。失败通知和
  Command Dock 状态提供面向用户的安全错误消息；失败的文本不会复制到剪贴板。
- 如果 ChatGPT Web 会话已过期，请在重试之前重新连接。如果 OpenAI API 密钥丢失或被拒绝，
  重试之前，请在提供商设置中更正它。

取消活动记录与失败的提交不同：取消会在捕获之前将其丢弃
准备或发送，因此该录音没有可重试的转录。

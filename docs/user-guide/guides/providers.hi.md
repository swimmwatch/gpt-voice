# एक ट्रांसक्रिप्शन प्रदाता चुनें और प्रबंधित करें

GPT-Voice एक समय में एक सक्रिय ट्रांसक्रिप्शन प्रदाता का उपयोग करता है। Command Dock में **Provider** चयनकर्ता से इसे चुनें। चयनकर्ता को स्विच करने से यह बदल जाता है कि कौन सा प्रदाता अगली रिकॉर्डिंग प्राप्त करेगा; यह प्रदाताओं के बीच सत्र, एपीआई कुंजी, बिलिंग या खाता पहुंच स्थानांतरित नहीं करता है।

| प्रदाता         | प्रमाणीकरण                                    | ऑडियो कहाँ भेजा जाता है                     |
| --------------- | --------------------------------------------- | ------------------------------------------- |
| **ChatGPT Web** | एक साइन-इन ChatGPT ब्राउज़र सत्र।             | साइन-इन ChatGPT सत्र के माध्यम से।          |
| **OpenAI API**  | आपके द्वारा प्रदान की गई एक OpenAI API कुंजी। | OpenAI का ऑडियो-ट्रांसक्रिप्शन समापन बिंदु। |

केवल उसी खाते का उपयोग करें जिसका उपयोग करने के लिए आप अधिकृत हैं। उपलब्धता, बिलिंग, कोटा, उपयोग सीमा और सेवा शर्तें प्रदाता खाते और सेवा द्वारा निर्धारित की जाती हैं। GPT-Voice उन्हें बायपास नहीं करता।

## ChatGPT Web

**ChatGPT Web** चुनें, फिर **Connect** चुनें। GPT-Voice ChatGPT पर एक ब्राउज़र लॉगिन विंडो खोलता है। वहां साइन-इन पूरा करें, फिर लॉगिन पेज बंद करें। GPT-Voice परिणामी ब्राउज़र सत्र को स्थानीय रूप से सहेजता है और बाद में लॉन्च होने पर अपना पृष्ठभूमि ब्राउज़र शुरू करता है जब सत्र अभी भी प्रयोग करने योग्य होता है।

जब सत्र तैयार हो जाता है, तो प्रदाता **Connected** दिखाता है। यदि सत्र समाप्त हो जाता है, तो GPT-Voice अप्रयुक्त संग्रहीत सत्र को हटा देता है और आपको **Connect** को फिर से चुनना होगा। एक प्रतिलेखन अनुरोध अपने अल्पकालिक एक्सेस टोकन को एक बार ताज़ा भी कर सकता है; जब अंतर्निहित सत्र अब मान्य नहीं है तो यह नए साइन-इन को प्रतिस्थापित नहीं करता है।

GPT-Voice से साइन आउट करने के लिए, प्रदाता सेटिंग खोलें और **Clear authentication** का उपयोग करें। यह GPT-Voice के सहेजे गए ChatGPT ब्राउज़र-सत्र डेटा और कैश्ड एक्सेस टोकन को हटा देता है। यह आपके खाते, सदस्यता, या अन्य ब्राउज़रों या उपकरणों द्वारा आयोजित किसी भी सत्र का प्रबंधन नहीं करता है।

## OpenAI API

**OpenAI API** चुनें और **Configure** चुनें, या चयनकर्ता के बगल में प्रदाता-सेटिंग्स नियंत्रण खोलें। अपनी स्वयं की एपीआई कुंजी चिपकाएँ और फ़ॉर्म सहेजें। GPT-Voice निश्चित प्रतिलेखन मॉडल का उपयोग करता है `whisper-1`; यह इस प्रदाता को कोई अन्य ट्रांसक्रिप्शन मॉडल चुनने की अनुमति नहीं देता है।

प्रदाता सेटिंग आपको यह भी चुनने देती है:

- **Language**: स्वचालित पहचान (डिफ़ॉल्ट), अंग्रेजी, रूसी, यूक्रेनी, या बेलारूसी।
- **Prompt**: प्रतिलेखन अनुरोध के साथ वैकल्पिक मार्गदर्शन भेजा गया।
- **Temperature**: `0` से `1` तक का मान; डिफ़ॉल्ट `0` है।

GPT-Voice एपीआई कुंजी को स्थानीय रूप से Electron safe storage के साथ संग्रहीत करता है जब वह सुरक्षा उपलब्ध होती है और इंटरफ़ेस में सहेजी गई कुंजी को कभी भी प्रदर्शित नहीं करती है। यदि सुरक्षित भंडारण अनुपलब्ध है, तो नई कुंजी को उस सुरक्षा के बिना संग्रहीत करने के बजाय सहेजना विफल हो जाता है। OpenAI API प्रदाता ब्राउज़र लॉगिन विंडो का उपयोग नहीं करता है।सहेजी गई एपीआई कुंजी को हटाने के लिए प्रदाता सेटिंग्स में **Clear authentication** का उपयोग करें। आपके द्वारा कॉन्फ़िगर की गई अगली कुंजी के लिए गैर-गुप्त ट्रांसक्रिप्शन विकल्प उपलब्ध रहते हैं, लेकिन प्रदाता तब तक कनेक्ट नहीं होता जब तक कि वैध कुंजी दोबारा सहेजी न जाए।

## किसी प्रदाता को बदलना या पुनर्प्राप्त करना

रिकॉर्डिंग शुरू करने से पहले आप प्रदाताओं को बदल सकते हैं। रिकॉर्ड करने से पहले पुष्टि करें कि नया चयनित प्रदाता **Connected** है; एक कॉन्फ़िगर किया गया ChatGPT Web सत्र और एक कॉन्फ़िगर किया गया OpenAI API कुंजी स्वतंत्र हैं।

यदि प्रतिलेखन विफल रहता है:

1. जांचें कि चयनित प्रदाता अभी भी जुड़ा हुआ है और उसका खाता ट्रांसक्रिप्शन का उपयोग कर सकता है।
2. **ChatGPT Web** के लिए, यदि सत्र समाप्त हो गया है या ब्राउज़र प्रारंभ नहीं हो सका है तो पुनः कनेक्ट करें।
3. **OpenAI API** के लिए, एपीआई कुंजी, बिलिंग या कोटा और प्रदाता की सेवा स्थिति की पुष्टि करें, फिर किसी भी आवश्यक सुधार को सहेजें।
4. नवीनतम तैयार किए गए कैप्चर को पुनः प्रयास करने के लिए, या एक नई रिकॉर्डिंग शुरू करने के लिए [रिकॉर्डिंग और ट्रांसक्रिप्शन](transcription.md) पर वापस लौटें।

प्रदाता सेटिंग्स भविष्य के अनुरोधों को प्रभावित करती हैं। वे रद्द की गई रिकॉर्डिंग को पुनर्प्राप्त नहीं कर सकते हैं या प्रदाता-पक्ष पहुंच, कोटा, नीति या सेवा त्रुटियों को ओवरराइड नहीं कर सकते हैं।

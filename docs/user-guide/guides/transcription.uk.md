# Записування й розшифрування

Перед записуванням підключіть постачальника зі станом **Connected** і надайте GPT-Voice доступ до мікрофона. Якщо ви
ще не завершили це налаштування, дивіться [перше використання](../getting-started.md), або перегляньте
[посібник постачальників](providers.md), щоб дізнатися про підключення й обліковий запис.

## Життєвий цикл запису

Почніть запис із Command Dock або налаштованим сполученням клавіш для записування (типове значення — `F9`). Щойно
захоплення мікрофона почнеться, стан зміниться з **Ready** на **Recording**. Під час записування основна дія —
**Stop recording** (типове значення — `F10`).

| Дія                 | Коли доступна                               | Що відбувається                                                                                         |
| ------------------- | ------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| **Start recording** | GPT-Voice не зайнятий.                      | Запитує доступ до мікрофона й починає нове захоплення. Нове захоплення очищує аудіо попередньої спроби. |
| **Pause**           | Захоплення записується.                     | Призупиняє поточне захоплення, не надсилаючи його.                                                      |
| **Resume**          | Захоплення призупинено.                     | Продовжує те саме захоплення.                                                                           |
| **Stop recording**  | Захоплення записується або призупинено.     | Завершує захоплення, готує аудіо й надсилає його вибраному постачальнику.                               |
| **Cancel**          | GPT-Voice починає, записує або призупинено. | Зупиняє й відкидає активне захоплення; воно не надсилається на розшифрування.                           |

Поки GPT-Voice зупиняє запис, готує аудіо, розшифровує або повторює спробу, дочекайтеся завершення поточної операції перед
початком нового запису. У цей час Command Dock показує стан оброблення.

## Що відбувається після зупинення

Після **Stop recording** GPT-Voice готує захоплене аудіо й показує **Transcribing**. Він надсилає підготовлене аудіо
постачальнику, вибраному в Command Dock:

- **ChatGPT Web** надсилає аудіо через сеанс браузера ChatGPT, у який ви ввійшли.
- **OpenAI API** надсилає аудіо до кінцевої точки розшифрування OpenAI, використовуючи налаштований ключ API.

Доступність постачальника, доступ до облікового запису, оплата, квоти та умови сервісу залежать від облікового запису
цього постачальника. GPT-Voice не обходить ці механізми контролю.

У разі успіху GPT-Voice копіює повернений текст до системного буфера обміну, змінює стан на **Copied to clipboard** і
надсилає запит на сповіщення про успіх. Вставте текст у програму, якою користувалися. GPT-Voice також зберігає текст,
ім'я постачальника та час запиту у локальній історії розшифровок; елементи керування історією описані окремо.

## Повторіть невдалу розшифровку

Після того як GPT-Voice підготував непорожнє захоплення, він зберігає підготовлене аудіо в пам'яті як найновіший запис,
доступний для повторної спроби. Якщо запит на розшифрування не вдався, використовуйте налаштовану дію повторної
розшифровки, коли GPT-Voice не зайнятий, щоб надіслати те саме підготовлене аудіо ще раз.

Повторна спроба не записує мікрофон повторно.

Ця копія для повторної спроби навмисно тимчасова:

- вона очищується перед початком нового запису;
- вона недоступна під час записування, оброблення або повторної спроби;
- вона зберігається лише в пам'яті запущеної програми; перезапуск GPT-Voice її видаляє.

Повторна спроба дає змогу повторити одне невдале надсилання після виправлення проблеми з підключенням, сеансом або
постачальником. Вона не змінює ліміти постачальника, не відновлює сеанс облікового запису, термін дії якого сплив, і
не гарантує, що постачальник прийме запит.

## Якщо записування або розшифрування не вдається

- **Could not access microphone** означає, що GPT-Voice не зміг отримати аудіопотік. Перевірте дозвіл на мікрофон у
  налаштуваннях конфіденційності операційної системи, переконайтеся, що мікрофон підключений, а потім почніть новий запис.
- **Transcription failed** означає, що постачальник повернув невдалий результат. Перед повторною спробою перевірте
  обліковий запис постачальника, мережу, доступність сервісу й застосовні ліміти.
- **Transcription error** означає, що GPT-Voice не зміг завершити підготовку або запит. Сповіщення про помилку та стан
  Command Dock містять безпечне повідомлення для користувача; невдалий текст не копіюється до буфера обміну.
- Якщо термін дії сеансу ChatGPT Web сплив, перепідключіть його перед повторною спробою. Якщо ключ OpenAI API відсутній
  або відхилений, виправте його в налаштуваннях постачальника перед повторною спробою.

Скасування активного запису відрізняється від невдалого надсилання: скасування відкидає захоплення перед підготовкою
або надсиланням, тому для такого запису немає розшифровки, доступної для повторної спроби.

# Выбор поставщика расшифровки и управление им

GPT-Voice одновременно использует одного активного поставщика расшифровки. Выберите его в селекторе **Provider** на
панели команд. Смена селектора изменяет поставщика для следующей записи; сеансы, API-ключи, оплата и доступ к учётной
записи между поставщиками не переносятся.

| Поставщик       | Аутентификация                        | Куда отправляется аудио                     |
| --------------- | ------------------------------------- | ------------------------------------------- |
| **ChatGPT Web** | Вошедший сеанс ChatGPT в браузере.    | Через вошедший сеанс ChatGPT.               |
| **OpenAI API**  | Предоставленный вами API-ключ OpenAI. | На конечную точку расшифровки аудио OpenAI. |

Используйте только учётную запись, которую вам разрешено использовать. Доступность, оплата, квоты, ограничения
использования и условия обслуживания определяются учётной записью и сервисом поставщика. GPT-Voice не обходит их.

## ChatGPT Web

Выберите **ChatGPT Web**, затем **Connect**. GPT-Voice откроет окно входа в ChatGPT в браузере. Завершите вход там,
после чего закройте страницу входа. GPT-Voice сохраняет полученный сеанс браузера локально и запускает фоновый
браузер при последующих запусках, когда сеанс ещё можно использовать.

Когда сеанс готов, поставщик показывает **Connected**. Если срок действия сеанса истёк, GPT-Voice удаляет
непригодный сохранённый сеанс, и нужно снова выбрать **Connect**. Запрос расшифровки также может один раз обновить
свой краткоживущий токен доступа; это не заменяет новый вход, если основной сеанс больше недействителен.

Чтобы выйти из GPT-Voice, откройте настройки поставщика и используйте **Clear authentication**. Это удаляет
сохранённые GPT-Voice данные сеанса ChatGPT в браузере и кэшированный токен доступа. Действие не управляет вашей
учётной записью, подпиской или сеансами в других браузерах и на других устройствах.

## OpenAI API

Выберите **OpenAI API** и **Configure** либо откройте элемент настроек поставщика рядом с селектором. Вставьте свой
API-ключ и сохраните форму. GPT-Voice использует фиксированную модель расшифровки `whisper-1`; для этого поставщика
нельзя выбрать другую модель расшифровки.

В настройках поставщика также можно выбрать:

- **Language**: автоматическое определение (по умолчанию), английский, русский, украинский или белорусский.
- **Prompt**: необязательную подсказку, отправляемую с запросом расшифровки.
- **Temperature**: значение от `0` до `1`; по умолчанию — `0`.

GPT-Voice хранит API-ключ локально через Electron safe storage, когда эта защита доступна, и никогда не показывает
сохранённый ключ в интерфейсе. Если защищённое хранилище недоступно, сохранение нового ключа завершается ошибкой, а
ключ не сохраняется без этой защиты. Поставщик OpenAI API не использует окно входа в браузере.

Используйте **Clear authentication** в настройках поставщика, чтобы удалить сохранённый API-ключ. Несекретные
параметры расшифровки останутся доступны для следующего настроенного ключа, но поставщик не подключён, пока снова не
будет сохранён действительный ключ.

## Смена поставщика или восстановление работы

Поставщика можно сменить перед началом записи. Перед записью убедитесь, что вновь выбранный поставщик имеет статус
**Connected**: настроенный сеанс ChatGPT Web и настроенный API-ключ OpenAI API независимы друг от друга.

Если расшифровка не удалась:

1. Убедитесь, что выбранный поставщик всё ещё подключён и его учётная запись может использовать расшифровку.
2. Для **ChatGPT Web** подключитесь снова, если сеанс истёк или браузер не удалось инициализировать.
3. Для **OpenAI API** проверьте API-ключ, оплату или квоту и состояние сервиса поставщика, затем сохраните нужное
   исправление.
4. Вернитесь к [записи и расшифровке](transcription.md), чтобы повторить последний подготовленный захват, или
   начните новую запись.

Настройки поставщика влияют на будущие запросы. Они не могут восстановить отменённую запись или обойти ошибки доступа,
квоты, политик либо сервиса на стороне поставщика.

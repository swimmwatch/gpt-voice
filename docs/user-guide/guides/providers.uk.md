# Вибір постачальника розшифрування та керування ним

GPT-Voice одночасно використовує одного активного постачальника розшифрування. Виберіть його в селекторі **Provider**
у Command Dock. Зміна селектора визначає, який постачальник отримає наступний запис; вона не передає сеанси, ключі
API, оплату чи доступ до облікового запису між постачальниками.

| Постачальник    | Автентифікація                             | Куди надсилається аудіо                       |
| --------------- | ------------------------------------------ | --------------------------------------------- |
| **ChatGPT Web** | Сеанс браузера ChatGPT, у який ви ввійшли. | Через сеанс ChatGPT, у який ви ввійшли.       |
| **OpenAI API**  | Наданий вами ключ OpenAI API.              | До кінцевої точки розшифрування аудіо OpenAI. |

Використовуйте лише обліковий запис, яким ви маєте право користуватися. Доступність, оплата, квоти, ліміти
використання й умови сервісу встановлюються обліковим записом і сервісом постачальника. GPT-Voice не обходить їх.

## ChatGPT Web

Виберіть **ChatGPT Web**, а потім **Connect**. GPT-Voice відкриє вікно входу в ChatGPT у браузері. Завершіть у ньому
вхід, а потім закрийте сторінку входу. GPT-Voice зберігає створений сеанс браузера локально й запускає фоновий
браузер під час наступних запусків, якщо сеанс ще придатний для використання.

Коли сеанс буде готовий, постачальник покаже **Connected**. Якщо термін дії сеансу спливе, GPT-Voice видалить
непридатний збережений сеанс, і вам доведеться знову вибрати **Connect**. Запит на розшифрування також може один раз
оновити його короткочасний токен доступу; це не замінює нового входу, якщо базовий сеанс більше не є дійсним.

Щоб вийти з GPT-Voice, відкрийте налаштування постачальника й скористайтеся **Clear authentication**. Це видаляє
збережені GPT-Voice дані сеансу ChatGPT у браузері та кешований токен доступу. Воно не керує вашим обліковим записом,
передплатою чи будь-якими сеансами в інших браузерах або на інших пристроях.

## OpenAI API

Виберіть **OpenAI API** і **Configure** або відкрийте елемент налаштувань постачальника поруч із селектором. Вставте
власний ключ API і збережіть форму. GPT-Voice використовує фіксовану модель розшифрування `whisper-1`; він не дає
цьому постачальнику вибрати іншу модель розшифрування.

Налаштування постачальника також дають змогу вибрати:

- **Language**: автоматичне визначення (типове значення), English, Russian, Ukrainian або Belarusian.
- **Prompt**: необов'язкові вказівки, що надсилаються із запитом на розшифрування.
- **Temperature**: значення від `0` до `1`; типове значення — `0`.

GPT-Voice зберігає ключ API локально через Electron safe storage, якщо цей захист доступний, і ніколи не показує
збережений ключ в інтерфейсі. Якщо захищене сховище недоступне, збереження нового ключа завершується помилкою замість
збереження без захисту. Постачальник OpenAI API не використовує вікно входу в браузері.

Скористайтеся **Clear authentication** у налаштуваннях постачальника, щоб видалити збережений ключ API. Несекретні
параметри розшифрування залишаються доступними для наступного налаштованого ключа, але постачальник не підключений,
доки дійсний ключ не буде збережено знову.

## Зміна або відновлення постачальника

Можна змінити постачальника перед початком запису. Перш ніж записувати, переконайтеся, що нововибраний постачальник
має стан **Connected**; налаштований сеанс ChatGPT Web і налаштований ключ OpenAI API є незалежними.

Якщо розшифрування не вдалося:

1. Перевірте, що вибраний постачальник усе ще підключений і його обліковий запис може користуватися розшифруванням.
2. Для **ChatGPT Web** перепідключіться, якщо сеанс сплив або не вдалося ініціалізувати браузер.
3. Для **OpenAI API** перевірте ключ API, оплату або квоту та стан сервісу постачальника, а потім збережіть потрібне
   виправлення.
4. Поверніться до [записування й розшифрування](transcription.md), щоб повторити останнє підготовлене захоплення,
   або почніть новий запис.

Налаштування постачальника впливають на майбутні запити. Вони не можуть відновити скасований запис або обійти
обмеження доступу, квоти, політики чи помилки сервісу на боці постачальника.

# 选择和管理转录提供商

GPT-Voice 一次使用一个活动转录提供程序。从 Command Dock 中的 **Provider** 选择器中选择它。切换选择器会改变接收下一个录音的提供商；它不会在提供商之间传输会话、API 密钥、计费或帐户访问权限。

| 供应商          | 认证                          | 音频发送到哪里            |
| --------------- | ----------------------------- | ------------------------- |
| **ChatGPT Web** | 已登录的 ChatGPT 浏览器会话。 | 通过登录的 ChatGPT 会话。 |
| **OpenAI API**  | 您提供的 OpenAI API 密钥。    | OpenAI 的音频转录端点。   |

仅使用您有权使用的帐户。可用性、计费、配额、使用限制和服务条款由提供商帐户和服务设置。 GPT-Voice 不会绕过它们。

## ChatGPT Web

选择 **ChatGPT Web**，然后选择 **Connect**。 GPT-Voice 在 ChatGPT 处打开浏览器登录窗口。在那里完成登录，然后关闭登录页面。 GPT-Voice 在本地保存生成的浏览器会话，并在会话仍然可用时在稍后启动时启动其后台浏览器。

当会话准备就绪时，提供程序会显示 **Connected**。如果会话过期，GPT-Voice 将删除不可用的存储会话，并且您必须再次选择 **Connect**。转录请求还可以刷新其短暂的访问令牌一次；当基础会话不再有效时，这不会取代新的登录。

要从 GPT-Voice 注销，请打开提供商设置并使用 **Clear authentication**。这将删除 GPT-Voice 保存的 ChatGPT 浏览器会话数据和缓存的访问令牌。它不管理您的帐户、订阅或其他浏览器或设备举行的任何会话。

## OpenAI API

选择 **OpenAI API** 并选择 **Configure**，或打开选择器旁边的提供商设置控件。粘贴您自己的 API 密钥并保存表单。 GPT-Voice使用固定转录模型`whisper-1`；它不允许该提供商选择其他转录模型。

提供商设置还允许您选择：

- **Language**：自动检测（默认）、英语、俄语、乌克兰语或白俄罗斯语。
- **Prompt**：随转录请求一起发送的可选指导。
- **Temperature**：从`0`到`1`的值；默认值为 `0`。

当该保护可用时，GPT-Voice 将 API 密钥与 Electron safe storage 一起存储在本地，并且从不在界面中显示保存的密钥。如果安全存储不可用，则保存新密钥会失败，而不是在没有这种保护的情况下存储它。 OpenAI API 提供程序不使用浏览器登录窗口。在提供商设置中使用 **Clear authentication** 删除保存的 API 密钥。非秘密转录选项对于您配置的下一个密钥仍然可用，但在再次保存有效密钥之前不会连接提供程序。

## 切换或恢复提供商

您可以在开始录制之前切换提供商。录制前确认新选择的提供商为**Connected**；配置的 ChatGPT Web 会话和配置的 OpenAI API 密钥是独立的。

如果转录失败：

1. 检查所选提供商是否仍处于连接状态，并且其帐户是否可以使用转录。
2. 对于**ChatGPT Web**，如果会话过期或浏览器无法初始化，请重新连接。
3. 对于 **OpenAI API**，确认 API 密钥、计费或配额以及提供商的服务状态，然后保存任何所需的更正。
4. 返回[录音和转录](transcription.md)重试最近准备的捕获，或开始新的录音。

提供商设置会影响未来的请求。他们无法恢复已取消的录制或覆盖提供商端访问、配额、策略或服务错误。

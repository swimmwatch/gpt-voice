# Канфідэнцыяльнасць і даныя

GPT-Voice працуе з маўленнем, вылучаным тэкстам, уліковымі дадзенымі і наладамі браўзера. На гэтай старонцы
апісваюцца бягучыя патокі даных і даступныя элементы кіравання для іх выдалення. Яна не замяняе палітыку
канфідэнцыяльнасці ці ўмовы сэрвісаў, якімі вы вырашылі карыстацца.

## Патокі даных

GPT-Voice адпраўляе даныя за межы вашага камп'ютара толькі тады, калі вы карыстаецеся функцыяй на аснове знешняга
сэрвісу:

| Функцыя                         | Адпраўленыя даныя                                     | Прызначэнне                                   |
| ------------------------------- | ----------------------------------------------------- | --------------------------------------------- |
| Расшыфроўка з **ChatGPT Web**   | Падрыхтаваны запіс                                    | ChatGPT праз сеанс браўзера, у які вы ўвайшлі |
| Расшыфроўка праз **OpenAI API** | Падрыхтаваны запіс і наладжаныя параметры расшыфроўкі | Канечная кропка OpenAI для расшыфроўкі аўдыя  |
| **Translate**                   | Вылучаны тэкст                                        | Google Translate                              |
| **Prettify**                    | Вылучаны тэкст і ваш наладжаны запыт Prettify         | Наладжаная канечная кропка Ollama або vLLM    |

Карыстайцеся толькі тымі ўліковымі запісамі і канечнымі кропкамі, якім давяраеце, і азнаёмцеся з іх умовамі
апрацоўкі даных. Лакальная канечная кропка Ollama або vLLM на loopback-інтэрфейсе пакідае запыт на камп'ютары, дзе
працуе сэрвіс; аддаленая канечная кропка атрымлівае тэкст. Для не-loopback канечнай кропкі Prettify выкарыстоўвайце
HTTPS. Сэрвісы ў браўзеры могуць карыстацца проксі, наладжаным у [наладах Network](settings/network.md).

GPT-Voice запісвае паспяховыя вынікі расшыфроўкі, перакладу і Prettify у сістэмны буфер абмену. Аперацыйная сістэма
і іншыя праграмы з доступам да буфера могуць захаваць ці прачытаць гэтае значэнне; ачысціце або замяніце яго пасля
ўстаўкі канфідэнцыяльнага выніку.

## Лакальныя даныя і часовая памяць

| Даныя                           | Дзе і як доўга захоўваюцца                                                                                                                                       |
| ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Гісторыя паспяховых расшыфровак | Лакальныя даныя SQLite у `gpt-voice.sqlite3`: час запыту, ідэнтыфікатар і імя пастаўшчыка, а таксама тэкст расшыфроўкі. Запісанае аўдыя не захоўваецца.          |
| Запіс для паўторнай спробы      | Найноўшае падрыхтаванае аўдыя захоўваецца толькі ў памяці запушчанай праграмы пасля няўдалай расшыфроўкі. Новы запіс або перазапуск GPT-Voice выдаляе яго.       |
| Кэш вынікаў Prettify            | Да 20 вынікаў захоўваюцца ў памяці да 60 секунд. Кантэкст кэша залежыць ад вылучанага тэксту, запыту і налад пастаўшчыка; пры выхадзе з GPT-Voice ён выдаляецца. |
| Налады і ідэнтычнасць браўзера  | Захоўваюцца ў лакальных даных GPT-Voice і выкарыстоўваюцца пры наступных запусках.                                                                               |
| Аўтэнтыфікацыя ChatGPT Web      | Лакальныя даныя сеанса і аўтэнтыфікацыі браўзера. Каб выдаліць іх, выкарыстоўвайце элемент кіравання пастаўшчыка.                                                |

Дзеянні з вылучаным тэкстам часова чытаюць і замяняюць змесціва буфера абмену, пакуль атрымліваюць вылучэнне. Калі
Translate або Prettify завяршаецца памылкай ці скасоўваецца, GPT-Voice аднаўляе значэнне буфера, захаванае перад
дзеяннем. У выпадку поспеху ў буферы застаецца вынік, які можна ўставіць.

## Уліковыя даныя і ўдакладненні пра шыфраванне

Ключы OpenAI API, ключы vLLM API і паролі HTTP/HTTPS-проксі захоўваюцца праз Electron safe storage, калі гэтая
абарона даступная. GPT-Voice не вяртае захаваныя значэнні ў вокны Settings. Калі абароненае сховішча недаступнае,
GPT-Voice не можа захаваць новы сакрэт праз гэты элемент кіравання.

Гэта не азначае, што кожны файл у каталогу даных праграмы зашыфраваны. У прыватнасці, гісторыя расшыфровак, звычайныя
налады і даныя сеанса ChatGPT Web маюць свае правілы лакальнага захоўвання. Не перадавайце ключы API, паролі проксі,
звесткі пра сеанс, надиктаваны тэкст або здымкі экрана з гэтымі данымі.

## Выдаленне або скід даных

Выберыце найбольш вузкі элемент кіравання, які адпавядае вашай патрэбе:

1. У акне History выкарыстоўвайце **Clear history**, каб назаўсёды выдаліць усе захаваныя запісы расшыфровак. Гл.
   [гісторыю і сістэмны трэй](guides/history-and-tray.md).
2. У элементах кіравання пастаўшчыком расшыфроўкі выкарыстоўвайце **Clear session** для ChatGPT Web або
   **Clear API key** для OpenAI API. Гл. [налады пастаўшчыка](settings/providers.md).
3. Калі гэта дастасоўна, выкарыстоўвайце **Clear API key** у [наладах Prettify](settings/prettify.md) і элемент
   **Clear** для пароля ў [наладах Network](settings/network.md).
4. Асобна замяніце або ачысціце сістэмны буфер абмену, калі ў ім застаўся вынік, які вы не хочаце мець для ўстаўкі.
5. Для поўнага лакальнага скіду выйдзіце з GPT-Voice праз трэй і выдаліце каталог даных праграмы, што захоўваецца:
   `%APPDATA%\GPT-Voice` у Windows або `~/.config/GPT-Voice` у Linux. Гэта выдаліць лакальныя налады, гісторыю і
   захаваныя даныя пастаўшчыкоў і запатрабуе паўторнай налады пасля пераўсталявання або запуску.

Выдаленне лакальных даных незваротнае. Звычайнае выдаленне праграмы наўмысна захоўвае гэтыя каталогі; інструкцыі для
кожнай платформы прыведзены ў раздзеле [ўсталяванне, абнаўленне або выдаленне](install.md).

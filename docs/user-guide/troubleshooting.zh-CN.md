# 故障排除

从可见的状态消息和您正在使用的功能开始。不要粘贴 API 密钥、代理密码、
ChatGPT 会话数据、口述文本、选定文本或将它们包含在支持请求中的屏幕截图。一个有用的
安全报告包括 GPT-Voice 版本、操作系统、软件包类型、功能、确切的非敏感消息以及
重现问题的步骤。

## 麦克风无法启动

如果 Command Dock 显示 **Could not access microphone**：

1. 确认麦克风已连接并且可供其他应用程序使用。
2. 在操作系统的隐私控制中允许 GPT-Voice 使用麦克风。
3. 关闭对设备具有独占控制权的其他软件，然后开始新的录制。
4. 如果在 GPT-Voice 打开时设备发生更改，请重新连接并重新启动 GPT-Voice，然后重试。

在 GPT-Voice 开始录制并且您将其停止之前，不会发送任何音频。对于录制控制和重试限制，
请参阅[记录和转录](guides/transcription.md)。

## ChatGPT Web 会话已断开连接

如果 **ChatGPT Web** 不是 **Connected**，则浏览器登录窗口不会完成，或者会话已过期：

1. 检查计算机是否可以访问 ChatGPT 并且该帐户是否有权使用该服务。
2. 选择**Connect**并在GPT-Voice浏览器窗口中完成登录。
3. 如果存储的会话不再有效，请使用**Clear session**确认并重新登录。
   4.如果浏览器初始化或连接仍然失败，请暂时不使用代理进行测试，然后检查浏览器
   和下面的网络设置。

清除会话会删除GPT-Voice保存的本地会话；它不会更改其他帐户或会话
浏览器。请参阅[提供商设置](settings/providers.md)。

## OpenAI API 转录失败

对于 **OpenAI API**，请检查是否已选择提供商并使用您自己的有效 API 密钥进行配置。确认
提供商帐户的账单、配额、使用限制和服务状态，然后保存所有更正并重试准备的内容
记录（如果仍然可用）。 GPT-Voice使用固定的`whisper-1`转录模型；没有型号
选择为此提供商进行修复。

将现有已保存的密钥排除在报告和屏幕截图之外。如需更换，输入新密钥并保存；使用
**Clear API key**当你想要GPT-Voice忘记它时。请参阅[提供商设置](settings/providers.md) 和
[提供商](guides/providers.md)。

## 美化无法到达模型

在 Prettify 可以处理所选文本之前，其提供者、端点和模型必须有效：

1. 在 **Settings** > **Prettify** 中，选择所需的提供商：Ollama 或 vLLM。
2. 确认本地或远程服务正在运行并且可从此计算机访问。
3. 检查提供商地址和型号名称，然后在所选型号尚未准备好时使用**Load model**。
4. 对于 vLLM，仅当端点需要时才提供 API 密钥。对于远程端点，使用 HTTPS 并确认您可以将所选文本发送到那里。
5. 再次运行快捷方式之前保存有效设置。

GPT-Voice 无法安装或操作 Ollama 或 vLLM。请参阅[美化设置](settings/prettify.md)字段
所选文本限制和取消的要求和[文本操作](guides/text-actions.md)。

## 代理或浏览器服务无法连接

ChatGPT Web 和翻译使用 GPT-Voice 浏览器上下文。如果任一服务在没有代理的情况下工作，但在使用代理时却无法工作：

1. 暂时关闭 **Proxy enabled** 并保存设置以将代理与服务隔离。
2. 重新启用时，请使用可访问的 `http://`、`https://` 或 `socks5://` 服务器 URL 并输入 HTTP/HTTPS
   在其专业领域的资历。
3. 删除 SOCKS5 凭据：CloakBrowser 不支持它们。
4. 如果启用 **GeoIP**，请记住它控制浏览器区域设置和时区。关闭GeoIP以测试保存的
   直接浏览器标识值。

对于浏览器运行时错误，请先在网络稳定后重试，然后按上述方法测试代理。设置**背景
browser** to **如果您在重现问题时需要观察浏览器上下文，则暂时可见**；恢复
之后是通常的 **Hidden** 设置。请参阅[浏览器设置](settings/browser.md)和
[网络设置](settings/network.md)。

## 快捷方式不运行

打开**Settings** > **Shortcuts**并确认该操作已启用，显示的快捷方式就是您按下的快捷方式，
并且更改已保存。 GPT-Voice 拒绝其自己的快捷方式之间的冲突，但另一个应用程序或
操作系统仍然可以保留相同的组合。

选择不同的快捷方式，保存它，然后在 GPT-Voice 空闲时重试。选定的文本操作也会等到
活动录制结束，翻译和美化不能同时运行。参见
[快捷方式设置](settings/shortcuts.md)。

## 剪贴板或选定的文本没有出现

对于转录，请在粘贴之前等待 **Copied to clipboard**。失败的转录不会被复制。对于
翻译或美化，在按操作快捷键之前在源应用程序中选择文本；美化接受
最多 16,000 个字符。

如果失败或取消，GPT-Voice 将恢复在选定文本操作之前捕获的剪贴板值。一个
成功结果替换剪贴板。确认源应用程序允许正常的复制操作；在Linux上，
当正常复制自动化失败时，GPT-Voice 还可以使用选择剪贴板。参见
[翻译并美化所选文本](guides/text-actions.md)。

## 安装、更新或启动问题

从官方 GitHub Release 中仅下载与您的平台匹配的包，并比较其 SHA-256 值
以及附带的校验和文件。再次运行 Windows 安装程序以更新现有安装。在 Linux 上，使用
deb 或 rpm 包的已记录的包管理器命令，或在运行 AppImage 之前使其可执行。签名和公证暂停时，没有受支持的 macOS 软件包。如果 GPT-Voice 在
安装或更新，重新启动计算机，重试已验证的包，并检查桌面运行时是否正常
依赖项可用。第一步不要删除保留的应用程序数据；它包含设置并保存
提供商数据。按照[安装、更新或删除](install.md)了解确切的软件包和删除过程。

## 如果问题仍然存在

尝试最小的安全复制：简短的非敏感录音、非敏感选定文本样本或
临时代理关闭检查。记录版本、平台、包、配置区域和可见错误，无需复制
凭据或私人内容。上述相关指南描述了支持的行为和可用的恢复
控制。

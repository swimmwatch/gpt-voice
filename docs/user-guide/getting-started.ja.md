# 最初の使用: プロバイダーに接続して音声を文字に起こす

[GPT-Voice](install.md) をインストールした後、1 つの文字起こしプロバイダーに接続し、マイクへのアクセスを許可し、
短い録音。 GPT-Voice は、成功した文字起こしをクリップボードにコピーします。自動的に入力されるわけではありません
使用していたアプリケーション。

## 1. 文字起こしプロバイダーを選択する

GPT-Voice を開き、コマンド ドックの **Provider** セレクターで **ChatGPT Web** または **OpenAI API** を選択します。

| プロバイダー    | 必要なもの                                                    | 初めてのアクション                                                         |
| --------------- | ------------------------------------------------------------- | -------------------------------------------------------------------------- |
| **ChatGPT Web** | サインインできる ChatGPT アカウント。                         | それを選択し、**Connect** を選択して、ブラウザーのサインインを完了します。 |
| **OpenAI API**  | 独自の OpenAI API キーと利用可能な API の請求または割り当て。 | それを選択し、**Configure** を選択してプロバイダー設定を開きます。         |

プロバイダーの可用性、制限、請求、および条件は、使用するアカウントによって制御されます。 GPT-Voice はバイパスしません
それらの限界。

### ChatGPT Web に接続する

1. **ChatGPT Web** を選択します。
2. **Connect** を選択します。
3. 開いたブラウザ ウィンドウで ChatGPT サインインを完了します。
4. ChatGPT の準備ができたら、ログイン ウィンドウを閉じて GPT-Voice に戻ります。

セッションの準備が完了すると、プロバイダーの状態は **Connected** に変わります。 GPT-Voice はブラウザー セッションを
ローカル アプリ データを保存し、その後の起動時にバックグラウンド ブラウザを自動的に起動します。セッションの有効期限が切れた場合は、
**Connect** をもう一度押して再度サインインします。

### OpenAI API を構成する

1. **OpenAI API** を選択します。
2. **Configure** を選択します。プロバイダー セレクターの横にあるプロバイダー設定コントロールを使用することもできます。
3. OpenAI API キーを貼り付けます。
4. 必要に応じて、転写言語、プロンプト、または温度を選択します。
5. **Save** を選択します。

転写モデルは `whisper-1` に固定されます。保存が成功すると、GPT-Voice はプロバイダーを次のように表示します。
**Connected** と、プロバイダーが構成されていることを報告します。アプリケーションは Electron を使用して API キーをローカルに保存します
利用可能な場合は安全に保管します。キーはインターフェイスに表示されません。 OpenAI API トランスクリプションでは、
ブラウザ。

## 2. マイクへのアクセスを許可する

最初の録音では、オペレーティング システムにマイクの許可が求められます。 GPT-Voice へのアクセスを許可してから、に戻ります。
コマンドドック。アクセスが拒否された場合、またはマイクが使用できない場合、ステータスには **エラー: アクセスできませんでした
マイク** があり、音声は送信されません。試す前に、オペレーティング システムのプライバシー コントロールで許可を有効にしてください。

## 3. 最初の録音を行う

1. **Connected** を示すプロバイダーを選択します。
2. **Start recording** を選択するか、表示された録音ショートカットを押します (デフォルトは `F9`)。
3. 短い文を話します。ステータスは **Recording** に変わります。
4. **Stop recording** (デフォルトは `F10`) を選択します。 GPT-Voice は、送信中にステータスを **Transcribing** に変更します。
   キャプチャした音声を選択したプロバイダーに送信します。
5. **Copied to clipboard** を待ってから、任意のテキスト フィールドに貼り付けて結果を確認します。

コマンド ドックからアクティブな記録を一時停止、再開、またはキャンセルすることもできます。キャンセルするとアクティブなものが破棄されます
文字起こしのために送信するのではなく、録音します。

## 最初の文字起こしが機能しない場合

- **Connected** が表示されない場合は、プロバイダー コントロールを再度開き、サインインを完了するか、有効な API キーを保存します。のためにChatGPT Web では、代わりにステータスがブラウザの初期化に失敗したか、セッションの有効期限が切れたことを示す場合があります。
- ステータスがマイク エラーを報告している場合は、GPT-Voice がオペレーティング システムでマイクを使用できるようにし、
  新しい録音。
- ステータスが **Transcription failed** または **Transcription error** と報告する場合は、選択したプロバイダーのアカウントを確認してください。
  接続と制限を確認してから、再試行してください。失敗した文字起こしはクリップボードにコピーされません。

# गोपनीयता और डेटा

GPT-Voice भाषण, चयनित पाठ, क्रेडेंशियल्स और ब्राउज़र सेटिंग्स को संभालता है। यह पृष्ठ वर्तमान डेटा पथों की व्याख्या करता है
और डेटा हटाने के लिए नियंत्रण उपलब्ध हैं। यह गोपनीयता नीति या आपके द्वारा चुनी गई किसी भी सेवा की शर्तों को प्रतिस्थापित नहीं करता है
उपयोग करें.

## डेटा प्रवाहित होता है

GPT-Voice आपके कंप्यूटर के बाहर डेटा तभी भेजता है जब आप किसी बाहरी सेवा द्वारा समर्थित सुविधा का उपयोग करते हैं:

| फ़ीचर                            | डेटा भेजा गया                                              | गंतव्य                                                          |
| -------------------------------- | ---------------------------------------------------------- | --------------------------------------------------------------- |
| **ChatGPT Web** के साथ प्रतिलेखन | तैयार रिकॉर्डिंग                                           | आपके साइन-इन किए हुए **ChatGPT Web** ब्राउज़र सत्र के माध्यम से |
| **OpenAI API** के साथ प्रतिलेखन  | तैयार रिकॉर्डिंग और कॉन्फ़िगर किए गए ट्रांसक्रिप्शन विकल्प | OpenAI का ऑडियो-ट्रांसक्रिप्शन समापन बिंदु                      |
| **Translate**                    | चयनित पाठ                                                  | Google Translate                                                |
| **Prettify**                     | चयनित पाठ और आपका कॉन्फ़िगर किया गया Prettify प्रॉम्प्ट    | आपका कॉन्फ़िगर किया गया Ollama या vLLM एंडपॉइंट                 |

उन खातों और अंतिम बिंदुओं का उपयोग करें जिन पर आप भरोसा करते हैं, और उनकी डेटा-हैंडलिंग शर्तों की समीक्षा करें। एक स्थानीय लूपबैक Ollama या vLLM समापन बिंदु
उस सेवा को चलाने वाली मशीन पर अनुरोध रखता है; एक दूरस्थ समापन बिंदु पाठ प्राप्त करता है। के लिए HTTPS का उपयोग करें
नॉन-लूपबैक प्रिटीफाई एंडपॉइंट। ब्राउज़र-आधारित सेवाएँ [नेटवर्क सेटिंग्स](settings/network.md). में कॉन्फ़िगर की गई प्रॉक्सी का उपयोग कर सकती हैं।

GPT-Voice सिस्टम क्लिपबोर्ड पर सफल ट्रांसक्रिप्शन, अनुवाद और प्रीटीफाई परिणाम लिखता है। परिचालन
सिस्टम और क्लिपबोर्ड एक्सेस वाले अन्य एप्लिकेशन उस मान को बनाए रख सकते हैं या पढ़ सकते हैं; चिपकाने के बाद इसे साफ़ करें या बदल दें
संवेदनशील आउटपुट.

## स्थानीय डेटा और अस्थायी मेमोरी

| डेटा                         | इसे कहाँ और कितने समय तक रखा जाता है                                                                                                                                                            |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| सफल प्रतिलेखन इतिहास         | अनुरोध समय, प्रदाता आईडी और नाम और ट्रांसक्रिप्शन टेक्स्ट के साथ `gpt-voice.sqlite3` पर स्थानीय SQLite डेटा। यह रिकॉर्ड किए गए ऑडियो को संग्रहीत नहीं करता है.                                  |
| पुनः प्रयास योग्य रिकॉर्डिंग | सबसे हालिया तैयार ऑडियो विफल ट्रांसक्रिप्शन के बाद केवल चल रहे एप्लिकेशन की मेमोरी में रखा जाता है। नई रिकॉर्डिंग शुरू करने या GPT-Voice को पुनः आरंभ करने से यह हट जाता है।                    |
| सुंदर परिणाम कैश             | अधिकतम 20 परिणाम 60 सेकंड तक स्मृति में रखे जाते हैं। कैश संदर्भ चयनित टेक्स्ट, प्रॉम्प्ट और कॉन्फ़िगर की गई प्रदाता सेटिंग्स पर निर्भर करता है; GPT-Voice बाहर निकलने पर इसे हटा दिया जाता है। |
| सेटिंग्स और ब्राउज़र पहचान   | GPT-Voice के स्थानीय एप्लिकेशन डेटा में संग्रहीत और बाद के लॉन्च के लिए उपयोग किया जाता है।                                                                                                     |
| ChatGPT Web प्रमाणीकरण       | स्थानीय ब्राउज़र-सत्र और प्रमाणीकरण डेटा। इसे साफ़ करने के लिए प्रदाता नियंत्रण का उपयोग करें।                                                                                                  |

चयनित-पाठ क्रियाएं चयन एकत्र करते समय क्लिपबोर्ड सामग्री को अस्थायी रूप से पढ़ती हैं और प्रतिस्थापित करती हैं। यदि अनुवाद याPrettify विफल हो जाता है या रद्द कर दिया जाता है, GPT-Voice कार्रवाई से पहले कैप्चर किए गए क्लिपबोर्ड मान को पुनर्स्थापित करता है। सफलता पर,
परिणाम आपके चिपकाने के लिए क्लिपबोर्ड पर रहता है।

## क्रेडेंशियल्स और एन्क्रिप्शन योग्यताएँ

OpenAI API कुंजियाँ, vLLM API कुंजियाँ, और HTTP/HTTPS प्रॉक्सी पासवर्ड Electron safe storage के माध्यम से संग्रहीत किए जाते हैं जब वह
सुरक्षा उपलब्ध है. GPT-Voice उन सहेजे गए मानों को उनके सेटिंग दृश्यों में वापस नहीं लौटाता है। यदि सुरक्षित भंडारण है
अनुपलब्ध, GPT-Voice उस नियंत्रण के माध्यम से कोई नया रहस्य सहेज नहीं सकता।

यह एप्लिकेशन-डेटा निर्देशिका में प्रत्येक फ़ाइल के लिए एक व्यापक एन्क्रिप्शन दावा नहीं है। विशेष रूप से, प्रतिलेखन
इतिहास, सामान्य सेटिंग्स और ChatGPT Web सत्र डेटा का अपना स्थानीय भंडारण व्यवहार होता है। एपीआई कुंजी साझा न करें,
प्रॉक्सी पासवर्ड, सत्र जानकारी, निर्देशित पाठ, या उनमें शामिल स्क्रीनशॉट।

## डेटा हटाएं या रीसेट करें

सबसे संकीर्ण नियंत्रण चुनें जो आपकी आवश्यकता को पूरा करता हो:

1. इतिहास विंडो में, सभी सहेजी गई ट्रांसक्रिप्शन प्रविष्टियों को स्थायी रूप से हटाने के लिए **Clear history** का उपयोग करें। देखिये
   [इतिहास और ट्रे](guides/history-and-tray.md).
2. ट्रांसक्रिप्शन प्रदाता नियंत्रण में, ChatGPT Web के लिए **Clear session** या OpenAI API के लिए **Clear API key** का उपयोग करें।
   [प्रदाता सेटिंग्स](settings/providers.md) देखें।
3. [सुंदर सेटिंग्स](settings/prettify.md), और पासवर्ड **Clear** नियंत्रण में **Clear API key** नियंत्रण का उपयोग करें
   [नेटवर्क सेटिंग्स](settings/network.md) में, जब लागू हो।
4. यदि सिस्टम क्लिपबोर्ड में वह आउटपुट है जिसे आप पेस्ट करने के लिए उपलब्ध नहीं कराना चाहते हैं तो उसे अलग से बदलें या साफ़ करें।
5. पूर्ण स्थानीय रीसेट के लिए, GPT-Voice को ट्रे से बाहर निकालें और इसकी बरकरार एप्लिकेशन-डेटा निर्देशिका को हटा दें:
   विंडोज़ पर `%APPDATA%\GPT-Voice` या लिनक्स पर `~/.config/GPT-Voice`। यह स्थानीय एप्लिकेशन-प्रबंधित को हटा देता है
   सेटिंग्स, इतिहास, और सहेजा गया प्रदाता डेटा, और पुनः स्थापित करने या पुनः लॉन्च करने के बाद फिर से सेटअप की आवश्यकता होती है।

स्थानीय डेटा को हटाना अपरिवर्तनीय है. अकेले एप्लिकेशन को अनइंस्टॉल करने से जानबूझकर वे निर्देशिकाएं बरकरार रहती हैं; देखिये
[प्लेटफ़ॉर्म-विशिष्ट अनइंस्टॉल व्यवहार के लिए ](install.md) इंस्टॉल करें, अपडेट करें या हटाएं।

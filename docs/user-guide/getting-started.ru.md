# Первый запуск: подключите поставщика и расшифруйте речь

После [установки GPT-Voice](install.md) подключите одного поставщика расшифровки, разрешите доступ к микрофону и
сделайте короткую запись. GPT-Voice копирует успешную расшифровку в буфер обмена; приложение не вводит текст
автоматически в другое приложение.

## 1. Выберите поставщика расшифровки

Откройте GPT-Voice и выберите **ChatGPT Web** или **OpenAI API** в селекторе **Provider** на панели команд.

| Поставщик       | Что требуется                                         | Действие при первом использовании                                      |
| --------------- | ----------------------------------------------------- | ---------------------------------------------------------------------- |
| **ChatGPT Web** | Учётная запись ChatGPT, в которую вы можете войти.    | Выберите его, затем **Connect** и завершите вход в браузере.           |
| **OpenAI API**  | Ваш API-ключ OpenAI и доступный лимит или оплата API. | Выберите его, затем **Configure**, чтобы открыть настройки поставщика. |

Доступность поставщика, лимиты, оплата и условия определяются используемой вами учётной записью. GPT-Voice не
обходит эти ограничения.

### Подключите ChatGPT Web

1. Выберите **ChatGPT Web**.
2. Нажмите **Connect**.
3. Выполните вход в ChatGPT в открывшемся окне браузера.
4. Когда ChatGPT будет готов, закройте окно входа и вернитесь в GPT-Voice.

Когда сеанс будет готов, состояние поставщика изменится на **Connected**. GPT-Voice сохраняет сеанс браузера в
локальных данных приложения и автоматически запускает фоновый браузер при следующих запусках. Если срок действия
сеанса истёк, снова нажмите **Connect** и войдите в систему.

### Настройте OpenAI API

1. Выберите **OpenAI API**.
2. Нажмите **Configure**. Также можно использовать элемент настроек поставщика рядом с селектором поставщика.
3. Вставьте свой API-ключ OpenAI.
4. При необходимости выберите язык расшифровки, подсказку или температуру.
5. Нажмите **Save**.

Для расшифровки используется фиксированная модель `whisper-1`. После успешного сохранения GPT-Voice показывает
состояние **Connected** и сообщает, что поставщик настроен. Приложение сохраняет API-ключ локально с помощью
безопасного хранилища Electron, когда оно доступно; ключ не отображается в интерфейсе. Расшифровка через OpenAI API
не использует браузер.

## 2. Разрешите доступ к микрофону

При первой записи операционная система запрашивает разрешение на использование микрофона. Разрешите доступ для
GPT-Voice, затем вернитесь к панели команд. Если доступ запрещён или микрофон недоступен, в состоянии отображается
**Error: Could not access microphone**, и аудио не отправляется. Включите разрешение в настройках конфиденциальности
операционной системы, затем повторите попытку.

## 3. Сделайте первую запись

1. Выберите поставщика со статусом **Connected**.
2. Нажмите **Start recording** или показанное сочетание клавиш записи (по умолчанию `F9`).
3. Произнесите короткую фразу. Состояние изменится на **Recording**.
4. Нажмите **Stop recording** (по умолчанию `F10`). Пока GPT-Voice отправляет захваченное аудио выбранному
   поставщику, состояние будет **Transcribing**.
5. Дождитесь **Copied to clipboard**, затем вставьте текст в любое поле, чтобы проверить результат.

Также можно приостановить, продолжить или отменить активную запись с панели команд. При отмене запись удаляется и не
отправляется на расшифровку.

## Если первая расшифровка не удалась

- Если **Connected** не отображается, снова откройте элементы управления поставщиком, завершите вход или сохраните
  действительный API-ключ. Для ChatGPT Web состояние может также сообщать, что не удалось инициализировать браузер
  или что срок действия сеанса истёк.
- Если состояние сообщает об ошибке микрофона, разрешите GPT-Voice использовать микрофон в операционной системе и
  создайте новую запись.
- Если состояние сообщает **Transcription failed** или **Transcription error**, проверьте учётную запись выбранного
  поставщика, подключение и лимиты, затем повторите попытку. Неудачная расшифровка не копируется в буфер обмена.

На следующих страницах руководства подробнее описаны управление записью, работа поставщиков, сочетания клавиш и
устранение неполадок. Пока что статус **Connected** у поставщика и **Copied to clipboard** подтверждают работу
основного пути.

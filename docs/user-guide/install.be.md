# Усталяванне, абнаўленне або выдаленне GPT-Voice

GPT-Voice мае гатовыя да працы выпускі для Windows і Linux. Вам не трэба асобна ўсталёўваць Node.js, npm, лакальную
мадэль, CUDA або Whisper.

Спампуйце пакет для свайго камп'ютара з [GitHub Releases](https://github.com/swimmwatch/gpt-voice/releases). Кожны
выпуск таксама змяшчае файлы `SHA256SUMS-*.txt` для адпаведных платформаў. Каб праверыць спампаванне, параўнайце
значэнне SHA-256, якое паказвае інструмент кантрольнай сумы вашай аперацыйнай сістэмы, з запісам для спампаванага
актыву выпуску.

> Выпускі для macOS прыпыненыя, пакуль рыхтуюцца падпісанне Developer ID і натарыяльнае засведчанне. Не ўсталёўвайце
> неафіцыйны DMG як GPT-Voice: у бягучых выпусках няма пакета macOS, які падтрымліваецца.

## Абярыце актыў выпуску

| Платформа              | Актыў выпуску            | Калі яго выкарыстоўваць                                                         |
| ---------------------- | ------------------------ | ------------------------------------------------------------------------------- |
| Windows                | `GPT-Voice Setup *.exe`  | Вам патрэбна звычайнае ўсталяванне Windows.                                     |
| Linux сямейства Debian | `gpt-voice_*_amd64.deb`  | Вы карыстаецеся Ubuntu, Debian, Linux Mint, Pop!_OS або падобным дыстрыбутывам. |
| Linux сямейства RPM    | `gpt-voice-*.x86_64.rpm` | Вы карыстаецеся Fedora, RHEL, CentOS, openSUSE або падобным дыстрыбутывам.      |
| Linux                  | `GPT-Voice-*.AppImage`   | Вам патрэбная партатыўная зборка без усталявання сістэмнага пакета.             |

## Windows

1. Спампуйце `GPT-Voice Setup *.exe` з апошняга выпуску і адкрыйце яго.
2. Калі будзе прапанавана, абярыце размяшчэнне ўсталявання.
3. Пакіньце ўключанымі цэтлікі на працоўным стале і ў меню Start, калі не хочаце запускаць праграму ўручную.
4. Завяршыце працу ўсталёўшчыка, затым адкрыйце **GPT-Voice** з меню Start, цэтліка на працоўным стале або апошняга
   экрана ўсталёўшчыка.

Пакет Windows — гэта ўсталёўшчык NSIS. Ён усталёўвае GPT-Voice, убудаванае асяроддзе браўзера, значкі, цэтлікі і
запіс дэўсталёўшчыка ў наладах Windows.

### Абнаўленне або выдаленне ў Windows

Каб абнавіць праграму, спампуйце навейшы `GPT-Voice Setup *.exe` і запусціце яго па-над існуючым усталяваннем.

Каб выдаліць праграму:

1. Адкрыйце **Settings** > **Apps** > **Installed apps**.
2. Знайдзіце **GPT-Voice**.
3. Абярыце **Uninstall**.

Выдаленне праграмы прыбірае ўсталяваныя файлы і цэтлікі, але наўмысна захоўвае вашы лакальныя налады і захаваны
сеанс пастаўшчыка ў `%APPDATA%\GPT-Voice`. Гэта дазваляе выкарыстоўваць іх пасля пераўсталявання. Выдаліце гэту
тэчку ўручную, толькі калі хочаце выдаліць і лакальныя даныя.

## Пакет Linux deb

Для Ubuntu, Debian, Linux Mint, Pop!_OS і падобных дыстрыбутываў усталюйце спампаваны пакет:

```bash
sudo apt install ./gpt-voice_*_amd64.deb
```

Калі ваша сістэма не можа ўсталяваць лакальны deb праз `apt`, выкарыстайце:

```bash
sudo dpkg -i ./gpt-voice_*_amd64.deb
sudo apt-get install -f
```

Пакет усталёўвае GPT-Voice у `/opt/GPT-Voice`, рэгіструе запускальнік і значкі на працоўным стале і дадае каманду
`gpt-voice`. Запускайце праграму з меню праграм або камандай `gpt-voice`.

Каб абнавіць праграму, усталюйце навейшы deb той жа камандай `apt install`. Каб выдаліць пакет, выкарыстайце
`sudo apt remove gpt-voice`; выкарыстайце `sudo apt purge gpt-voice`, калі таксама хочаце выдаліць канфігурацыю
пакета.

## Пакет Linux rpm

Для Fedora, RHEL, CentOS, openSUSE і падобных дыстрыбутываў выкарыстоўвайце менеджар пакетаў свайго дыстрыбутыва,
каб ён мог разабраць залежнасці пакета. Не выкарыстоўвайце звычайны `rpm -i` для стандартнага ўсталявання.

```bash
# Fedora, RHEL, CentOS і сумяшчальныя дыстрыбутывы
sudo dnf install ./gpt-voice-*.x86_64.rpm

# Старэйшыя сістэмы CentOS або RHEL
sudo yum install ./gpt-voice-*.x86_64.rpm

# openSUSE
sudo zypper install ./gpt-voice-*.x86_64.rpm
```

Пакет rpm усталёўвае той жа запускальнік, значкі і каманду `gpt-voice`, што і пакет deb. Яго актыў выпуску прызначаны
для настольных сістэм `x86_64`. У мінімальнай устаноўцы Linux перад усталяваннем уключыце звычайныя рэпазітарыі
працоўнага стала і асяроддзя выканання, каб менеджар пакетаў мог атрымаць залежнасці.

Каб абнавіць праграму, усталюйце навейшы rpm той жа камандай менеджара пакетаў. Каб выдаліць яго, выкарыстоўвайце
`sudo dnf remove gpt-voice` у Fedora/RHEL/CentOS або `sudo zypper remove gpt-voice` у openSUSE.

## Linux AppImage

Выкарыстоўвайце AppImage, калі аддаяце перавагу партатыўнай копіі, а не сістэмнаму пакету.

1. Спампуйце `GPT-Voice-*.AppImage`.
2. Зрабіце файл выканальным і запусціце яго:

   ```bash
   chmod +x GPT-Voice-*.AppImage
   ./GPT-Voice-*.AppImage
   ```

Пры першым запуску GPT-Voice, калі магчыма, рэгіструе для вашага карыстальніка лакальны запускальнік і значок на
працоўным стале. Каб абнавіць праграму, спампуйце навейшы AppImage, зрабіце яго выканальным і запусціце замест
старога файла.

Каб выдаліць усталяванне AppImage, выйдзіце з GPT-Voice, запусціце з гэтага AppImage каманду выдалення інтэграцыі з
працоўным сталом, а затым выдаліце файл:

```bash
./GPT-Voice-*.AppImage --remove-linux-appimage-desktop-integration
```

## Захаваныя даныя Linux

Выдаленне пакета deb або rpm, а таксама выдаленне AppImage не выдаляе вашы налады ці захаваны сеанс пастаўшчыка. Яны
застаюцца ў `~/.config/GPT-Voice`. Выдаліце гэтую тэчку ўручную, толькі калі хочаце атрымаць чысты скід.

Калі ўсталяванне завершана, перайдзіце да [першага запуску](getting-started.md), каб падключыць пастаўшчыка
расшыфроўкі і зрабіць першы запіс.

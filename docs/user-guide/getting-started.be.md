# Першы запуск: падключыце пастаўшчыка і расшыфруйце маўленне

Пасля [ўсталявання GPT-Voice](install.md) падключыце аднаго пастаўшчыка расшыфроўкі, дазвольце доступ да мікрафона і
зрабіце кароткі запіс. GPT-Voice капіруе паспяховую расшыфроўку ў буфер абмену; ён не набірае тэкст аўтаматычна ў
праграме, якой вы карысталіся.

## 1. Абярыце пастаўшчыка расшыфроўкі

Адкрыйце GPT-Voice і абярыце **ChatGPT Web** або **OpenAI API** у селектары **Provider** у Command Dock.

| Пастаўшчык      | Што вам патрэбна                                             | Дзеянне пры першым выкарыстанні                                   |
| --------------- | ------------------------------------------------------------ | ----------------------------------------------------------------- |
| **ChatGPT Web** | Уліковы запіс ChatGPT, у які вы можаце ўвайсці.              | Абярыце яго, затым **Connect** і завершыце ўваход у браўзеры.     |
| **OpenAI API**  | Ваш уласны ключ OpenAI API і даступная аплата або квота API. | Абярыце яго, затым **Configure**, каб адкрыць налады пастаўшчыка. |

Даступнасць пастаўшчыкоў, абмежаванні, аплата і ўмовы кантралююцца ўліковым запісам, які вы выкарыстоўваеце.
GPT-Voice не абыходзіць гэтыя абмежаванні.

### Падключыце ChatGPT Web

1. Абярыце **ChatGPT Web**.
2. Абярыце **Connect**.
3. Завяршыце ўваход у ChatGPT у акне браўзера, якое адкрыецца.
4. Калі ChatGPT будзе гатовы, зачыніце гэта акно ўваходу і вярніцеся ў GPT-Voice.

Калі сеанс будзе гатовы, стан пастаўшчыка зменіцца на **Connected**. GPT-Voice захоўвае сеанс браўзера ў вашых
лакальных даных праграмы і аўтаматычна запускае фонавы браўзер пры наступных запусках. Калі сеанс скончыцца,
зноў абярыце **Connect**, каб увайсці яшчэ раз.

### Наладзьце OpenAI API

1. Абярыце **OpenAI API**.
2. Абярыце **Configure**. Можна таксама выкарыстаць элемент наладаў пастаўшчыка побач з селектарам пастаўшчыка.
3. Устаўце свой ключ OpenAI API.
4. Пры жаданні абярыце мову расшыфроўкі, запыт або тэмпературу.
5. Абярыце **Save**.

Мадэль расшыфроўкі фіксаваная: `whisper-1`. Пасля паспяховага захавання GPT-Voice паказвае пастаўшчыка як
**Connected** і паведамляе, што ён наладжаны. Праграма захоўвае ключ API лакальна праз Electron safe storage, калі
яно даступнае; ключ не паказваецца зноў у інтэрфейсе. Расшыфроўка праз OpenAI API не выкарыстоўвае браўзер.

## 2. Дазвольце доступ да мікрафона

Пры першым запісе аперацыйная сістэма запытае дазвол на выкарыстанне мікрафона. Дазвольце доступ для GPT-Voice,
затым вярніцеся ў Command Dock. Калі доступ забаронены або мікрафон недаступны, стан паказвае **Error: Could not
access microphone**, і аўдыя не адпраўляецца. Перад наступнай спробай уключыце дазвол у наладах канфідэнцыяльнасці
сваёй аперацыйнай сістэмы.

## 3. Зрабіце першы запіс

1. Абярыце пастаўшчыка са станам **Connected**.
2. Абярыце **Start recording** або націсніце паказанае спалучэнне клавіш для запісу (па змаўчанні `F9`).
3. Скажыце кароткі сказ. Стан зменіцца на **Recording**.
4. Абярыце **Stop recording** (па змаўчанні `F10`). GPT-Voice зменіць стан на **Transcribing**, калі адправіць
   захопленае аўдыя выбранаму пастаўшчыку.
5. Пачакайце **Copied to clipboard**, затым устаўце тэкст у любое тэкставае поле, каб пацвердзіць вынік.

Вы таксама можаце прыпыніць, аднавіць або адмяніць актыўны запіс з Command Dock. Адмена адкідвае актыўны запіс замест
адпраўкі яго на расшыфроўку.

## Калі першая расшыфроўка не працуе

- Калі не паказваецца **Connected**, зноў адкрыйце элементы кіравання пастаўшчыком і завяршыце ўваход або захавайце
  сапраўдны ключ API. Для ChatGPT Web стан можа замест гэтага паведамляць, што не атрымалася ініцыялізацыя браўзера
  або што сеанс скончыўся.
- Калі стан паказвае памылку мікрафона, дазвольце GPT-Voice выкарыстоўваць мікрафон у аперацыйнай сістэме і зрабіце
  новы запіс.
- Калі стан паказвае **Transcription failed** або **Transcription error**, праверце ўліковы запіс, падключэнне і
  абмежаванні выбранага пастаўшчыка, затым паспрабуйце зноў. Няўдалая расшыфроўка не капіруецца ў буфер абмену.

Наступныя старонкі кіраўніцтва больш падрабязна апісваюць кіраванне запісам, паводзіны пастаўшчыка, спалучэнні
клавіш і вырашэнне праблем. Пакуль што пастаўшчык са станам **Connected** і вынік **Copied to clipboard** пацвярджаюць,
што базавы шлях працуе.

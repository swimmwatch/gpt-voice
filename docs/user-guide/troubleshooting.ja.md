# トラブルシューティング

まずは、表示されるステータス メッセージと使用していた機能から始めます。 API キー、プロキシ パスワード、
ChatGPT セッション データ、口述テキスト、選択したテキスト、またはそれらを含むスクリーンショットをサポート リクエストに含めます。便利な
安全レポートには、GPT-Voice バージョン、オペレーティング システム、パッケージ タイプ、機能、正確な非機密メッセージ、および
問題を再現する手順。

## マイクを開始できません

Command Dock に **Could not access microphone** が表示される場合:

1. マイクが接続されており、他のアプリケーションで使用できることを確認します。
2. GPT-Voice がオペレーティング システムのプライバシー コントロールでマイクを使用できるようにします。
3. デバイスを排他的に制御する他のソフトウェアを閉じて、新しい記録を開始します。
4. GPT-Voice が開いている間にデバイスが変更された場合は、再試行する前にデバイスを再接続し、GPT-Voice を再起動します。

GPT-Voice が録音を開始し、あなたが録音を停止するまで、音声は送信されません。記録制御と再試行制限については、
[記録と転写](guides/transcription.md)を参照してください。

3. 保存されたセッションが有効でなくなった場合は、**Clear session** を使用して確認し、再度サインインします。
4. ブラウザの初期化または接続がまだ失敗する場合は、プロキシを使用せずに一時的にテストし、ブラウザの状態を確認します。

## Prettify がモデルにアクセスできません

Prettify が選択したテキストを処理できるようにするには、そのプロバイダー、エンドポイント、モデルが有効である必要があります。

1. **Settings** > **Prettify** で、目的のプロバイダーを選択します: Ollama または vLLM。
2. ローカルまたはリモート サービスが実行中であり、このコンピュータからアクセスできることを確認します。
3. プロバイダーのアドレスとモデル名を確認し、選択したモデルが準備できない場合は **Load model** を使用します。
4. vLLM の場合は、エンドポイントで API キーが必要な場合にのみ API キーを指定します。リモート エンドポイントの場合は、HTTPS を使用して確認します選択したテキストをそこに送信することが許可されていることがわかります。
5. ショートカットを再度実行する前に、有効な設定を保存します。

GPT-Voice は、Ollama または vLLM をインストールまたは操作しません。フィールドについては [Prettify settings](settings/prettify.md)] を参照してください。
要件と、選択したテキストの制限とキャンセルのための [テキスト アクション](guides/text-actions.md)]。

## ショートカットが実行されない

**Settings** > **Shortcuts** を開き、アクションが有効になっていること、表示されたショートカットが押したものであることを確認します。
そして変更が保存されました。 GPT-Voice は、自身のショートカットと、別のアプリケーションまたは
オペレーティング システムは引き続き同じ組み合わせを予約できます。

別のショートカットを選択して保存し、GPT-Voice がアイドル状態のときに再試行してください。選択されたテキストのアクションも、
アクティブな録音が終了すると、Translation と Prettify を同時に実行できなくなります。参照
[ショートカット設定](settings/shortcuts.md)。

## クリップボードまたは選択したテキストが表示されませんでした

文字起こしの場合は、**Copied to clipboard** 待ってから貼り付けてください。失敗した文字起こしはコピーされません。のために
Translation または Prettify では、アクションのショートカットを押す前にソース アプリケーションでテキストを選択します。プリティファイは受け入れます
最大 16,000 文字。

失敗またはキャンセルの場合、GPT-Voice は選択テキスト アクションの前にキャプチャしたクリップボード値を復元します。あ
成功した結果はクリップボードを置き換えます。ソース アプリケーションが通常のコピー操作を許可していることを確認します。 Linux では、
GPT-Voice は、通常のコピー自動化が失敗した場合に選択クリップボードを使用することもできます。参照
[選択したテキストを翻訳して整形します](guides/text-actions.md)。
